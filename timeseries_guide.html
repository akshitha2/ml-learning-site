The Big Picture

Why does the order matter?

Time series modelling is not plug-and-play. Each step reveals something about your data that directly informs the next step. Skip a step and you risk building a model on faulty foundations.

01

Collect & Visualise EDA

Plot the raw series. Look for trend, seasonality, outliers, missing values. Get a gut feeling.

02

Decompose Trend · Season · Residual

Formally split the series into its components. Understand what kind of non-stationarity exists.

03

Test for Unit Root ADF Test

Statistically confirm whether differencing is needed. This gives you d.

04

ACF & PACF p and q

Read the correlation plots on the stationary series to find AR order (p) and MA order (q).

05

Fit Model ARIMA / ARIMAx

Build ARIMA(p,d,q) or ARIMAx if external variables drive the cycles.

06

Validate & Forecast Residual Check

Residuals should be white noise. Then generate future predictions with confidence intervals.

Step 01

Exploratory Data Analysis

Before writing a single line of modelling code, understand your data visually and statistically.

What to look for when plotting

📈

Trend

Is the series moving consistently upward or downward over time? Revenue going from ₹48k to ₹120k over 3 years = clear upward trend.

🔄

Seasonality

Does a pattern repeat at fixed intervals? December revenue spikes every year = seasonality with period m=12.

⚡

Outliers / Shocks

Any sudden spikes or drops? COVID lockdown month = massive negative shock that should be noted.

📊

Variance

Does the spread around the trend grow over time? If yes → multiplicative decomposition, consider log transform.

Time Series Decomposition

Formally split the series into four components to understand the structure before modelling.

Python — Decomposition 
from statsmodels.tsa.seasonal import seasonal_decompose

# model='additive'  → if variance is CONSTANT across time
# model='multiplicative' → if variance GROWS with the level
result = seasonal_decompose(y, model='additive', period=12)

fig, axes = plt.subplots(4, 1, figsize=(12, 10))

result.observed .plot(ax=axes[0], title='Observed',   color='#e8eaf0')
result.trend    .plot(ax=axes[1], title='Trend',      color='#e8c547')
result.seasonal .plot(ax=axes[2], title='Seasonality', color='#5b9cf6')
result.resid    .plot(ax=axes[3], title='Residual',   color='#56d9a0')

plt.tight_layout()
plt.show()

# Save residuals for ACF/PACF later
residuals_decomp = result.resid.dropna()

Step 03

Augmented Dickey-Fuller Test

Visually we suspect a unit root. Now we confirm it statistically. This gives us the value of d.

ADF Test — What it fits under the hood ΔY_t = α + δ·Y_{t-1} + β₁·ΔY_{t-1} + β₂·ΔY_{t-2} + ... + ε_t H₀: δ = 0 → Unit root EXISTS → Non-stationary H₁: δ < 0 → No unit root → Stationary Decision rule: p-value < 0.05 → Reject H₀ → Stationary ✅ p-value > 0.05 → Fail to reject H₀ → Unit Root ❌
Python — ADF Test 
from statsmodels.tsa.stattools import adfuller

def adf_test(series, name=''):
    result = adfuller(series, autolag='AIC')
    print(f"\n── ADF Test: {name} ──────────────")
    print(f"  ADF Statistic : {result[0]:.4f}")
    print(f"  p-value       : {result[1]:.4f}")
    print(f"  Critical Values:")
    for key, val in result[4].items():
        print(f"    {key}: {val:.4f}")
    if result[1] < 0.05:
        print("  ✅ STATIONARY (reject H₀)")
    else:
        print("  ❌ UNIT ROOT (fail to reject H₀)")

# ── Test raw series ────────────────────────────
adf_test(y, 'Raw Revenue')
# Expected: p-value > 0.05 → UNIT ROOT

# ── Test first difference ──────────────────────
adf_test(y.diff().dropna(), 'First Difference ΔRevenue')
# Expected: p-value < 0.05 → STATIONARY ✅
# This confirms d = 1

# ── Also test X variable ──────────────────────
adf_test(X['consumer_confidence'], 'Consumer Confidence')
adf_test(X['consumer_confidence'].diff().dropna(), 'ΔConsumer Confidence')

Our ADF Results

Series TestedADF Statp-valueVerdictAction
Raw Revenue (Y)-1.230.661 Unit Root ❌ Difference once
ΔRevenue (Y differenced)-5.810.000 Stationary ✅ d = 1 confirmed

ΔConsumer Confidence-6.120.000 Stationary ✅ Use differenced X in ARIMAx
✅
Conclusion from ADF Both Y and X are I(1) — they have unit roots and become stationary after one round of differencing. Therefore d = 1. In ARIMAx, we will use differenced X as the exogenous variable.

Step 04

ACF & PACF — Finding p and q

Now that we know d=1, we difference the series and look at ACF/PACF to find the AR order (p) and MA order (q).

📉

PACF → gives p (AR order)

PACF cuts off sharply after lag p. Count how many lags are significant before the cutoff. That number is p.

📉

ACF → gives q (MA order)

ACF cuts off sharply after lag q. Count how many lags are significant before the cutoff. That number is q.

Python — ACF & PACF 
from statsmodels.graphics.tsaplots import plot_acf, plot_pacf

y_diff = y.diff().dropna()

fig, axes = plt.subplots(1, 2, figsize=(14, 5))

# ACF  → for MA order (q)
# lags=20 means we inspect up to lag 20
plot_acf(y_diff, lags=20, ax=axes[0],
         title='ACF — ΔRevenue  (→ tells q)')

# PACF → for AR order (p)
plot_pacf(y_diff, lags=20, ax=axes[1],
          title='PACF — ΔRevenue  (→ tells p)')

plt.tight_layout()
plt.show()

# ── For ARIMAx: pre-whiten first ──────────────
from sklearn.linear_model import LinearRegression

X_diff = X.diff().dropna()
y_diff_aligned = y_diff[X_diff.index]

reg = LinearRegression().fit(X_diff, y_diff_aligned)
clean_resid = y_diff_aligned - reg.predict(X_diff)

# These ACF/PACF plots are more accurate for ARIMAx
plot_acf (clean_resid, lags=20, title='ACF  (pre-whitened) → q')
plot_pacf(clean_resid, lags=20, title='PACF (pre-whitened) → p')
plt.show()

Reading the plots — pattern guide

ACF PatternPACF PatternModel Suggested
Cuts off after lag qDecays graduallyMA(q) → set p=0, q=that lag
Decays graduallyCuts off after lag pAR(p) → set p=that lag, q=0
Both decay graduallyBoth decay graduallyARMA(p,q) → both non-zero
Significant at lag 12Significant at lag 12Seasonality → consider SARIMA
📌
Our Result PACF cuts off after lag 1 → p = 1. ACF cuts off after lag 1 → q = 1. Combined with d = 1 from ADF, our starting model is ARIMA(1, 1, 1). We will confirm with AIC.

Step 05A

Fitting ARIMA(p, d, q)

We now fit the model and validate it. We also use AIC/BIC grid search to confirm our p, q choices.

ARIMA(1,1,1) — What it models ΔRevenue_t = c + φ₁·ΔRevenue_{t-1} + ε_t + θ₁·ε_{t-1} where: φ₁ = AR coefficient (how much last month's change matters) θ₁ = MA coefficient (how much last month's shock lingers) ε_t = random shock this month
Python — ARIMA Fit + AIC Grid Search 
from statsmodels.tsa.arima.model import ARIMA
import warnings
warnings.filterwarnings('ignore')

# ── Grid search for best p, q ─────────────────
best_aic = float('inf')
best_order = None
results_table = []

for p in range(0, 4):
    for q in range(0, 4):
        try:
            model = ARIMA(y, order=(p, 1, q))
            res   = model.fit()
            results_table.append({
                'order': (p, 1, q),
                'AIC'  : round(res.aic, 2),
                'BIC'  : round(res.bic, 2)
            })
            if res.aic < best_aic:
                best_aic   = res.aic
                best_order = (p, 1, q)
        except:
            continue

print(f"Best order: {best_order}  AIC: {best_aic:.2f}")

# ── Fit the best model ────────────────────────
final_model = ARIMA(y, order=best_order).fit()
print(final_model.summary())

# ── Residual diagnostics ──────────────────────
residuals = final_model.resid

fig, axes = plt.subplots(2, 2, figsize=(12, 8))
residuals.plot(ax=axes[0,0], title='Residuals over time')
residuals.plot(kind='kde', ax=axes[0,1], title='Residual distribution')
plot_acf(residuals, ax=axes[1,0], title='ACF of Residuals')
plot_pacf(residuals, ax=axes[1,1], title='PACF of Residuals')
# Goal: No significant spikes in ACF/PACF of residuals
# That would mean all patterns are captured ✅
plt.show()

Residual Validation — What to check

✅

Good Residual Signs

• Random scatter around zero
• No visible pattern over time
• Normal bell curve shape
• ACF/PACF all within confidence bands

❌

Bad Residual Signs

• Trend still visible
• Funnel shape (changing variance)
• Significant spikes in ACF/PACF
• Heavy tails in distribution

Step 05B

Fitting ARIMAx with External Variables

Since decomposition showed a cyclical component tied to Consumer Confidence, we upgrade to ARIMAx — ARIMA with an exogenous variable.

ARIMAx(1,1,1) — Model equation ΔRevenue_t = c + φ₁·ΔRevenue_{t-1} + θ₁·ε_{t-1} + β₁·ΔConsumerConfidence_t + ε_t β₁ tells us: "For every 1 point rise in Consumer Confidence, Revenue changes by β₁ thousand rupees, after accounting for its own time series pattern"
Python — ARIMAx Full Workflow 
from statsmodels.tsa.statespace.sarimax import SARIMAX

# ── Prepare data ──────────────────────────────
X_exog = df[['consumer_confidence']]  # raw — SARIMAX handles d internally

# ── Fit ARIMAx(1,1,1) with exogenous variable ─
arimax_model = SARIMAX(
    y,
    exog  = X_exog,
    order = (1, 1, 1),   # p=1, d=1, q=1
    trend = 'c'           # include constant
)
arimax_result = arimax_model.fit(disp=False)
print(arimax_result.summary())

# ── Check coefficient significance ────────────
# Look at p-values in the summary table
# consumer_confidence p-value < 0.05 → variable is significant ✅
# consumer_confidence p-value > 0.05 → remove it, use plain ARIMA

# ── Compare ARIMA vs ARIMAx using AIC ─────────
print(f"ARIMA   AIC: {final_model.aic:.2f}")
print(f"ARIMAx  AIC: {arimax_result.aic:.2f}")
# Lower AIC = better model
# If ARIMAx AIC is lower → external variable helps ✅
# If ARIMA  AIC is lower → external variable not useful ❌

# ── Residual check for ARIMAx ─────────────────
arimax_resid = arimax_result.resid
plot_acf(arimax_resid, lags=20)
# No spikes → model captured all patterns ✅

ARIMA vs ARIMAx — When to use which?

SituationUseWhy
No external drivers identifiedARIMASimpler, fewer parameters
Clear external driver (economic index, weather)ARIMAxExternal variable explains cyclical component
ARIMAx β has p-value > 0.05ARIMAExternal variable doesn't help — remove it

Step 06

Generating the Forecast

Final step — use the validated model to predict future revenue with confidence intervals.

Python — Forecast 
# ── ARIMA forecast (no external variable) ─────
forecast_steps = 6  # predict next 6 months

forecast_arima = final_model.get_forecast(steps=forecast_steps)
forecast_mean  = forecast_arima.predicted_mean
conf_int       = forecast_arima.conf_int(alpha=0.05)  # 95% confidence

print(pd.DataFrame({
    'Forecast'   : forecast_mean,
    'Lower 95%'  : conf_int.iloc[:, 0],
    'Upper 95%'  : conf_int.iloc[:, 1]
}))

# ── ARIMAx forecast (needs future X values) ───
# You need projected Consumer Confidence for next 6 months
# This can come from a government forecast or your own estimate
X_future = pd.DataFrame({
    'consumer_confidence': [78, 80, 79, 82, 85, 84]
})

forecast_arimax = arimax_result.get_forecast(
    steps = forecast_steps,
    exog  = X_future
)
arimax_mean = forecast_arimax.predicted_mean
arimax_ci   = forecast_arimax.conf_int()

# ── Plot both forecasts ───────────────────────
plt.figure(figsize=(13, 6))
plt.plot(y, label='Actual Revenue', color='#e8eaf0')
plt.plot(forecast_mean,  label='ARIMA Forecast',  color='#5b9cf6')
plt.plot(arimax_mean,    label='ARIMAx Forecast', color='#e8c547')
plt.fill_between(arimax_ci.index,
                 arimax_ci.iloc[:,0], arimax_ci.iloc[:,1],
                 alpha=0.2, color='#e8c547', label='95% CI')
plt.legend()
plt.title('Revenue Forecast — ARIMA vs ARIMAx')
plt.show()

Quick Reference

Model Selection Decision Tree

Use this flowchart every time you start a new time series problem.

Q1. Plot the series. Does it have a clear upward/downward trend?
YES → Likely non-stationary. Run ADF test.
NO → Could be stationary. Run ADF test to confirm.

Q2. ADF test p-value > 0.05?
YES → Unit root confirmed → d = 1. Difference once and re-test.
NO → Stationary → d = 0. Skip differencing.

Q3. Does decomposition show strong repeating seasonal pattern?
YES → Use SARIMA(p,d,q)(P,D,Q)m with seasonal terms.
NO → Plain ARIMA or ARIMAx is sufficient.

Q4. Do you have external variables that drive the series?
YES → Test their stationarity. If significant in model → use ARIMAx.
NO → Use plain ARIMA.

Q5. Read PACF of stationary series → p. Read ACF → q.
Confirm with AIC/BIC grid search across p ∈ [0,3], q ∈ [0,3].

Q6. Fit model. Check residuals. ACF/PACF of residuals all within bands?
YES → Model is good. Proceed to forecast ✅
NO → Patterns remain. Try higher p or q, or add seasonal terms.

The Golden Rules of Time Series Modelling

Never skip these steps, in this order.

1. Always plot first 2. Always test stationarity 3. Never ignore unit roots 4. ACF/PACF on stationary series only 5. Always validate residuals 6. Use AIC to compare models 7. External variables → ARIMAx 8. Seasonality → SARIMA
