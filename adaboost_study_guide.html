AdaBoost

Adaptive Boosting — turning weak learners into a strong ensemble through sequential correction and weighted voting

01 · INTUITION

The Core Idea

AdaBoost answers one question: can a team of barely-better-than-random classifiers collectively become a strong classifier?

Yes — if they learn from each other's mistakes. After each weak learner makes errors, those misclassified points are made harder to ignore by increasing their weights. Every subsequent learner is forced to focus on the cases that stumped its predecessors.

The Bouncer Analogy

Imagine a club with bad bouncers — each can only check one thing (age, height, jacket colour). No single bouncer is reliable. But if each bouncer passes the hard cases forward to the next one, and the final decision is a weighted vote where more accurate bouncers get louder say — the team becomes surprisingly effective.

1

Round 1 — First Stump

All points have equal weight. Stump finds the best split. Gets some points wrong.

2

Reweight

Wrong points get upweighted. Correct points get downweighted. The mistake becomes the priority.

3

Round 2 — Next Stump

Trains on reweighted data. Forced to focus on what Round 1 missed. Makes different mistakes.

4

Repeat T rounds

Each stump specialises in whoever the previous stumps struggled with. Hard cases bubble forward.

5

Weighted Vote

Final answer = sign(Σ αₜ · hₜ(x)). Stumps with lower weighted error get higher α — louder vote.

Key Insight

Each stump only needs to do slightly better than random (ε < 0.5) on the weighted data. The theory guarantees that training error decreases exponentially with T as long as this condition holds.

02 · ALGORITHM

Full Algorithm

Binary classification with y ∈ {−1, +1} (this sign convention is essential for the math to work cleanly).

AdaBoost Algorithm — Complete Pseudocode
INITIALISE: w_i = 1/N for all i = 1...N

FOR t = 1, 2, ..., T:

  1. Train weak learner h_t(x) on weighted dataset
     → exhaustive split search across all features & thresholds
     → weighted impurity: misclassifying heavy points costs more

  2. Compute weighted error:
     ε_t = Σ w_i · 1[h_t(x_i) ≠ y_i]    (sum over all i)

  3. Compute tree weight:
     α_t = ½ · ln((1 - ε_t) / ε_t)

  4. Update sample weights:
     w_i ← w_i · exp(−α_t · y_i · h_t(x_i))

  5. Renormalise: w_i ← w_i / Σ w_j

FINAL PREDICTION:
  H(x) = sign(Σ α_t · h_t(x))    (sum over t = 1...T)

Why y ∈ {−1, +1} and not {0, 1}?

The product y · h(x) is the mathematical heart of both the weight update and the final prediction. With {−1, +1}:

  • Correct prediction: same sign → y · h(x) = +1
  • Wrong prediction: opposite sign → y · h(x) = −1

With {0, 1}, a wrong prediction gives 0 · 1 = 0 or 1 · 0 = 0 — you lose the sign information entirely. The weight update formula would break.

03 · FORMULAS

Every Formula Explained

Formula 1 — Weighted Error ε

Weighted Error
ε_t = Σᵢ wᵢ · 𝟙[hₜ(xᵢ) ≠ yᵢ]
Sum the weights of all misclassified points. Since weights sum to 1, ε ∈ [0, 1].
This is not just a count of mistakes — a mistake on a heavy point (weight 0.6) contributes far more than a mistake on a light point (weight 0.05).
ε = 0.5 means the stump is no better than random guessing on the weighted data.

Formula 2 — Tree Weight α

Tree Weight (Vote Strength)
αₜ = ½ · ln((1 − εₜ) / εₜ)
The fraction (1−ε)/ε = (fraction correct) / (fraction wrong).
When mostly correct: ratio is large → ln is large → α is large (loud vote).
When 50/50: ratio = 1 → ln(1) = 0 → α = 0 (no vote at all).
When mostly wrong: ratio < 1 → ln is negative → α is negative (vote gets flipped — a consistently wrong stump is still useful).

α values at key ε points

ε (weighted error) Interpretation α value Effect
0.0 Perfect stump +∞ Infinite trust
0.1 90% accurate ≈ +1.10 Strong vote
0.3 70% accurate ≈ +0.42 Moderate vote
0.5 Random guess 0 No vote
0.7 Consistently wrong ≈ −0.42 Vote flipped
1.0 Always wrong −∞ Perfect (inverted)

Formula 3 — Weight Update

Sample Weight Update
wᵢ(t+1) = wᵢ(t) · exp(−αₜ · yᵢ · hₜ(xᵢ))
The product yᵢ · hₜ(xᵢ) is the clever trick:
→ Correct prediction: yᵢ · hₜ(xᵢ) = +1 → exponent = −α → multiply by e^(−α) < 1 → weight DECREASES
→ Wrong prediction: yᵢ · hₜ(xᵢ) = −1 → exponent = +α → multiply by e^(+α) > 1 → weight INCREASES

The magnitude of change scales with α — a confident stump's mistakes are taken more seriously. After updating all weights, renormalise so they sum to 1.

Formula 4 — Final Prediction

Final Ensemble Prediction
H(x) = sign(Σₜ αₜ · hₜ(x))
Weighted sum of all stump predictions. Each stump votes (+1 or −1), scaled by its α.
If the weighted sum is positive → predict +1. If negative → predict −1.
This is a linear combination in "decision space" — each stump carves out a region, α determines how much it matters.

Formula 5 — Training Error Bound

Theoretical Guarantee
Training Error ≤ exp(−2γ²T)
Where γ = minimum "edge" each stump has over random (γ = 0.5 − ε_max).
Training error decreases exponentially with the number of rounds T.
Condition: every stump must achieve ε_t ≤ 0.5 − γ for some fixed γ > 0.
This is why weak learners are sufficient — even a tiny edge compounds into a strong guarantee.

Round 2 — Stump 2

Stump 2 trains on reweighted data. With B and E each at weight 0.25, it searches for a threshold that handles them.

Say Stump 2 finds: "If X ≤ 0.9, predict −1. Else predict +1." — this correctly catches E (X=0.8).

Final Prediction Example

Final Vote for a New Point x = 1.2

Say after 3 rounds: α₁=0.203, α₂=0.150, α₃=0.421

  • h₁(1.2) = +1 → contribution: 0.203 × (+1) = +0.203
  • h₂(1.2) = +1 → contribution: 0.150 × (+1) = +0.150
  • h₃(1.2) = −1 → contribution: 0.421 × (−1) = −0.421
  • Sum = 0.203 + 0.150 − 0.421 = −0.068

sign(−0.068) = −1 → predict class −1

Notice: Tree 3 had the highest α (0.421) so its vote for −1 overrides the two +1 votes from Trees 1 and 2.

06 · PITFALLS

What Can Go Wrong

Pitfall 1: Outlier Compounding

The most dangerous failure mode. A mislabelled or noisy point that can never be correctly classified gets upweighted every single round — because no stump can get it right.

Compounding Chain
Round 1: Outlier misclassified → weight ↑ to 0.35
Round 2: Stump obsesses over outlier → gets it right
         but now misclassifies A, B → their weights ↑
Round 3: Stump focuses on A, B → outlier misclassified again
         → weight ↑ to 0.55
Round 5: Outlier weight = 0.70 → dominates the entire dataset
         Every stump warps around one garbage point
Root Cause

AdaBoost uses exponential loss L = Σ exp(−y·F(x)). This function grows extremely fast for confidently wrong predictions, amplifying outlier weights without bound.

Pitfall 2: Overfitting in Later Rounds

Early rounds: hard cases are genuine patterns. Late rounds: the only remaining hard cases are pure noise. AdaBoost doesn't distinguish — it keeps fitting.

Fix: early stopping — monitor validation error, stop when it stops improving for K rounds.

Pitfall 3: ε > 0.5

If a stump achieves weighted error > 0.5, α becomes negative. The stump's vote gets automatically flipped (consistently wrong → vote reversed). In practice, if this happens frequently, the algorithm becomes unstable. The theoretical guarantee requires ε < 0.5 − γ for all rounds.

Key Rule

Noisy / mislabelled data → Random Forest. Outliers get diluted across 500 independent trees. AdaBoost compounds them.

07 · COMPARISON

AdaBoost vs Random Forest

AdaBoost

  • Sequential — each tree depends on previous
  • Decision stumps (max_depth=1)
  • Full dataset, reweighted each round
  • Weighted votes — α based on accuracy
  • Reduces bias primarily
  • Sensitive to outliers (compounding)
  • Can overfit with more trees
  • Cannot enforce monotonic constraints

Random Forest

  • Parallel — all trees independent
  • Deep trees (low bias)
  • Bootstrap samples (~63% per tree)
  • Equal votes — all trees count the same
  • Reduces variance primarily
  • Robust to outliers (dilution)
  • More trees never overfits — plateaus
  • Cannot enforce monotonic constraints

AdaBoost as Special Case of GBM

Every boosting algorithm is doing gradient descent in function space. The difference is which loss function they minimise:

AdaBoost
L = Σ exp(−y · F(x))
Exponential loss. Very sensitive to outliers. Weight reweighting = gradient of this loss.
GBM (Classification)
L = Σ log(1 + e^(−y·F(x)))
Log-loss. More robust. Residuals = gradient of log-loss.
GBM (Regression)
L = Σ (yᵢ − F(xᵢ))²
MSE loss. Residuals are literal differences yᵢ − F(xᵢ).
The Connection

AdaBoost's sample reweighting mechanism is mathematically equivalent to fitting residuals of exponential loss. The two seemingly different approaches — reweighting points vs fitting residuals — are the same algorithm under different loss functions.

08 · HYPERPARAMETERS

sklearn AdaBoostClassifier

from sklearn.ensemble import AdaBoostClassifier

clf = AdaBoostClassifier(
    n_estimators=50,          # number of boosting rounds
    learning_rate=1.0,        # shrinks each stump's contribution
    estimator=None,           # default: DecisionTreeClassifier(max_depth=1)
    algorithm='SAMME',        # SAMME or SAMME.R
    random_state=42
)
n_estimators
default = 50
Number of boosting rounds (stumps). More rounds → lower training error, but risk of overfitting on noisy data. Unlike Random Forest, more is NOT always better. Use cross-validation to find the plateau. Typical range: 50–500.
learning_rate
default = 1.0
Shrinks each stump's contribution: final sum becomes Σ η·αₜ·hₜ(x). Lower η = smaller steps = need more stumps to compensate. Standard recipe: learning_rate=0.1 + high n_estimators + early stopping. Direct tradeoff: halve η → roughly double n_estimators needed.
estimator
default = stump (depth=1)
The weak learner. Default is DecisionTreeClassifier(max_depth=1). Can increase to max_depth=2 for slightly stronger learners. Going too deep breaks the "weak learner" guarantee and overfits faster. Stumps are the standard choice.
algorithm
default = 'SAMME'
SAMME: discrete boosting, uses class predictions (0/1 per stump). SAMME.R: uses predicted class probabilities instead — generally converges faster and performs better. SAMME.R is almost always the better choice in practice.

Tuning Order

1. Start: n_estimators=50, learning_rate=1.0, max_depth=1 (defaults)
2. Reduce learning_rate to 0.1
3. Increase n_estimators to compensate — use CV to find plateau
4. If still overfitting → check for noisy data, consider RF instead
5. If underfitting → try max_depth=2 on base estimator

Multiclass: SAMME Algorithm

Standard AdaBoost is binary only. For K classes, sklearn uses SAMME with a modified α:

SAMME Tree Weight (Multiclass)
αₜ = ½ · ln((1 − εₜ) / εₜ) + ln(K − 1)
The extra ln(K−1) term adjusts for the fact that random guessing gives ε = (K−1)/K, not 0.5.
With K=2: ln(2−1) = ln(1) = 0, so it reduces to standard binary AdaBoost exactly.
09 · REGRESSION

AdaBoost for Regression

sklearn provides AdaBoostRegressor. The core idea is identical — sequential stumps, reweight hard points — but "hard" now means large prediction error, not misclassified.

Key Differences from Classifier

Three Changes

1. Continuous Error per Point

eᵢ = |yᵢ − h(xᵢ)| / max_j |yⱼ − h(xⱼ)|
Normalised to [0,1]. Worst predicted point always gets eᵢ = 1.

2. Different Loss Functions Available

loss parameterError functionUse case
linear (default)|error|Standard
squareerror²Penalise large errors more
exponential1 − exp(−|error|)More robust to outliers

3. Different α and Weight Update Formulas

The formulas change but the intuition is identical — better stumps get more trust, harder points get more weight.

In Practice

AdaBoostRegressor is rarely used. GBM, XGBoost, and LightGBM handle regression more naturally and almost always outperform it. Know it exists; don't prioritise it.
