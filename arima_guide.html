рҹ“җ Stationarity вҖ” The Foundation of Everything

A time series is stationary if its statistical properties (mean, variance, autocorrelation) do not change over time. Almost every classical forecasting model requires stationarity.

Formal Definition (Weak / Covariance Stationarity)
A series Yt is weakly stationary if:
1. E[Yt] = Ој вҖ” constant mean
2. Var(Yt) = ПғВІ вҖ” constant variance
3. Cov(Yt, YtвҲ’k) = Оі(k) вҖ” depends only on lag k, not on t

Types of Non-Stationarity

TypeDescriptionFix
TrendMean grows/shrinks over timeDifferencing (d вүҘ 1)
HeteroscedasticityVariance changes over timeLog transform, Box-Cox
SeasonalityPeriodic patterns repeatSeasonal differencing (D вүҘ 1)
Structural BreakSudden regime changeDummy variables, regime models
Unit RootStochastic trend (random walk)Differencing d times

рҹ”ў Differencing вҖ” The Math

1st Difference: О”y_t = y_t вҲ’ y_{tвҲ’1} 2nd Difference: О”ВІy_t = О”y_t вҲ’ О”y_{tвҲ’1} = y_t вҲ’ 2y_{tвҲ’1} + y_{tвҲ’2} Seasonal Diff (period s): О”_s y_t = y_t вҲ’ y_{tвҲ’s} Backshift Operator B: BВ·y_t = y_{tвҲ’1} вҶ’ О”y_t = (1вҲ’B)y_t вҶ’ О”^d y_t = (1вҲ’B)^d y_t
Numerical Example
y = [100, 102, 106, 111, 117, 122]

1st Diff (О”y): 102вҲ’100=2, 106вҲ’102=4, 111вҲ’106=5, 117вҲ’111=6, 122вҲ’117=5
2nd Diff (О”ВІy): 4вҲ’2=2, 5вҲ’4=1, 6вҲ’5=1, 5вҲ’6=вҲ’1

The 2nd difference removes the accelerating trend. Series now fluctuates around 0.

Visual: Non-Stationary vs Stationary After Differencing

NON-STATIONARY: y=[100,102,106,111,117,122,128,134,140,145]
AFTER 1st DIFF: О”y=[2,4,5,6,5,6,6,6,5]

How Many Times to Difference? (Choosing d)

Run the ADF test (Tab в‘Ј) after each differencing. If p-value > 0.05, difference again. Usually d = 0, 1, or 2. Over-differencing introduces unnecessary MA terms вҖ” stop as soon as ADF confirms stationarity.

вҡ пёҸ Over-differencing Warning
If you difference more than needed, you artificially introduce autocorrelation into the series. d=2 should only be used if d=1 still fails the ADF test. Always verify visually too.

рҹ“Ҡ Partial AutoCorrelation Function (PACF)

PACF measures the correlation between y_t and y_{tвҲ’k} after removing the effect of all intermediate lags (1 through kвҲ’1). It isolates the direct relationship of lag k only.

PACF(k) = Corr(y_t, y_{tвҲ’k} | y_{tвҲ’1}, y_{tвҲ’2}, ..., y_{tвҲ’k+1}) Computed via Yule-Walker equations (Levinson-Durbin recursion): PACF(1) = ПҶвӮҒвӮҒ = ПҒ(1) [no intermediate lags to remove] PACF(2) = ПҶвӮӮвӮӮ = [ПҒ(2) вҲ’ ПҒ(1)ВІ] / [1 вҲ’ ПҒ(1)ВІ] For lag k: regress y_t on y_{tвҲ’1},...,y_{tвҲ’k} ПҶ_kk = coefficient on y_{tвҲ’k} (last coefficient)

рҹ”ў Step-by-Step Calculation

Same series: y = [3,5,4,6,5,7,6,8,7,9]  |  ПҒ(1) = 0.3000  |  ПҒ(2) = 0.5333
PACF(1) = ПҒ(1) = 0.3000 (Same as ACF at lag 1 вҖ” no intermediates to control for) PACF(2) = [ПҒ(2) вҲ’ ПҒ(1)ВІ] / [1 вҲ’ ПҒ(1)ВІ] = [0.5333 вҲ’ 0.3000ВІ] / [1 вҲ’ 0.3000ВІ] = [0.5333 вҲ’ 0.0900] / [1 вҲ’ 0.0900] = 0.4433 / 0.9100 вүҲ 0.4871 Interpretation: After removing the lag-1 effect, lag-2 still has a direct correlation of 0.487 with the current value.

рҹ§Ә Augmented Dickey-Fuller (ADF) Test

The ADF test formally checks whether a unit root exists in a time series. A unit root = non-stationary (series behaves like a random walk).

Hypotheses:
HвӮҖ (Null): Unit root exists вҶ’ series is NON-stationary
HвӮҒ (Alternative): No unit root вҶ’ series IS stationary

p-value < 0.05 вҶ’ Reject HвӮҖ вҶ’ STATIONARY вң“
p-value вүҘ 0.05 вҶ’ Fail to reject HвӮҖ вҶ’ NON-STATIONARY вҶ’ difference!

The Math вҖ” Where Does the Test Come From?

Start with AR(1): y_t = ПҶВ·y_{tвҲ’1} + Оө_t Subtract y_{tвҲ’1}: О”y_t = (ПҶвҲ’1)В·y_{tвҲ’1} + Оө_t Let Оҙ = ПҶ вҲ’ 1: О”y_t = ОҙВ·y_{tвҲ’1} + Оө_t HӮҖ: Оҙ = 0 (ПҶ=1 вҶ’ unit root вҶ’ non-stationary вҶ’ О”y_t = Оө_t, pure noise) HӮҒ: Оҙ < 0 (ПҶ<1 вҶ’ stationary вҶ’ series mean-reverts) AUGMENTED version вҖ” adds lagged differences to remove autocorrelation in residuals (p lags chosen via AIC/BIC): О”y_t = Оұ + ОІВ·t + ОҙВ·y_{tвҲ’1} + ОЈвұј ОівұјВ·О”y_{tвҲ’j} + Оө_t вҶ‘ вҶ‘ вҶ‘ const trend вҶҗ this is the test term Test statistic: П„ = ОҙМӮ / SE(ОҙМӮ) (NOT standard t-distribution!)

3 Variants вҖ” Which to Use?

VariantEquation IncludesUse When
No constant, no trendonly ОҙВ·y_{tвҲ’1}Series fluctuates around 0
Constant, no trendОұ + ОҙВ·y_{tвҲ’1}Series has non-zero mean (most common)
Constant + trendОұ + ОІt + ОҙВ·y_{tвҲ’1}Series has a deterministic trend

рҹ”ў Numerical Walkthrough

Random Walk (non-stationary): y_t = y_{tвҲ’1} + Оө_t
y = [0, 0.5, 0.8, 0.3, 0.9, 1.4, 1.1, 1.7, 2.1, 1.8]
О”y = [0.5, 0.3, вҲ’0.5, 0.6, 0.5, вҲ’0.3, 0.6, 0.4, вҲ’0.3]

Regress О”y_t on y_{tвҲ’1}: expect Оҙ вүҲ 0 вҶ’ t-stat near 0 вҶ’ p-value > 0.05
вҶ’ Cannot reject HвӮҖ вҶ’ NON-STATIONARY вҶ’ apply differencing
Stationary AR(1), ПҶ=0.5: y_t = 0.5В·y_{tвҲ’1} + Оө_t
Оҙ = ПҶ вҲ’ 1 = 0.5 вҲ’ 1 = вҲ’0.5
This negative value is significantly different from 0 вҶ’ t-stat is very negative
вҶ’ Compare to MacKinnon critical value (e.g. вҲ’2.86 at 5%)

Python Code

from statsmodels.tsa.stattools import adfuller result = adfuller(y, autolag='AIC') # AIC selects lag length p print(f"ADF Statistic : {result[0]:.4f}") print(f"p-value : {result[1]:.4f}") print(f"Lags Used : {result[2]}") print(f"Critical Values:") for k, v in result[4].items(): print(f" {k}: {v:.3f}") # Interpret if result[1] < 0.05: print("вҶ’ STATIONARY (reject HвӮҖ)") else: print("вҶ’ NON-STATIONARY вҶ’ apply differencing!") # Also run KPSS from statsmodels.tsa.stattools import kpss stat, p_val, lags, crit = kpss(y, regression='c') print(f"KPSS p-value: {p_val:.4f}") # KPSS: p < 0.05 вҶ’ reject HвӮҖ of stationarity вҶ’ non-stationary

вҡҷпёҸ ARIMA(p, d, q) вҖ” AutoRegressive Integrated Moving Average

ARIMA(p, d, q) = AR(p) + I(d) + MA(q) p = order of AutoRegressive part d = degree of differencing (makes series stationary) q = order of Moving Average part Full ARIMA equation on differenced series О”бөҲy_t: О”бөҲy_t = c + ПҶвӮҒО”бөҲy_{tвҲ’1} + ... + ПҶвӮҡО”бөҲy_{tвҲ’p} + Оө_t + ОёвӮҒОө_{tвҲ’1} + ... + Оё_qОө_{tвҲ’q} Using backshift operator B (BВ·y_t = y_{tвҲ’1}): ПҶ(B)(1вҲ’B)бөҲ y_t = c + Оё(B)Оө_t where ПҶ(B) = 1 вҲ’ ПҶвӮҒB вҲ’ ПҶвӮӮBВІ вҲ’ ... вҲ’ ПҶвӮҡBбө– (AR polynomial) Оё(B) = 1 + ОёвӮҒB + ОёвӮӮBВІ + ... + Оё_qBбөҚ (MA polynomial)

в‘  AR(p) вҖ” AutoRegressive Model

y_t = c + ПҶвӮҒy_{tвҲ’1} + ПҶвӮӮy_{tвҲ’2} + ... + ПҶвӮҡy_{tвҲ’p} + Оө_t Оө_t ~ N(0, ПғВІ) (white noise, IID) Stationarity condition: all roots of ПҶ(B)=0 lie OUTSIDE the unit circle For AR(1): |ПҶвӮҒ| < 1
рҹ”ў AR(1) Walkthrough: y_t = 0.7В·y_{tвҲ’1} + Оө_t, yвӮҖ = 2
Оө = [0.5, вҲ’0.3, 0.8, вҲ’0.2, 0.6]

yвӮҒ = 0.7 Г— 2.0000 + 0.5000 = 1.9000
yвӮӮ = 0.7 Г— 1.9000 + (вҲ’0.3000) = 1.3300 вҲ’ 0.3000 = 1.0300
yвӮғ = 0.7 Г— 1.0300 + 0.8000 = 0.7210 + 0.8000 = 1.5210
yвӮ„ = 0.7 Г— 1.5210 + (вҲ’0.2000) = 1.0647 вҲ’ 0.2000 = 0.8647
yвӮ… = 0.7 Г— 0.8647 + 0.6000 = 0.6053 + 0.6000 = 1.2053
Key AR Properties:
|ПҶ| < 1 вҶ’ stationary (mean-reverts to Ој/(1вҲ’ПҶвӮҒ))
ПҶ = 1 вҶ’ unit root вҶ’ random walk (non-stationary, needs differencing)
|ПҶ| > 1 вҶ’ explosive (grows without bound)
Larger |ПҶ| вҶ’ stronger memory, slower geometric decay in ACF

в‘Ў MA(q) вҖ” Moving Average Model

y_t = Ој + Оө_t + ОёвӮҒОө_{tвҲ’1} + ОёвӮӮОө_{tвҲ’2} + ... + Оё_qОө_{tвҲ’q} Always stationary regardless of Оё values! Invertibility condition: all roots of Оё(B)=0 lie OUTSIDE unit circle For MA(1): |ОёвӮҒ| < 1 Memory: exactly q lags (finite memory)
рҹ”ў MA(1) Walkthrough: y_t = Оө_t + 0.5В·Оө_{tвҲ’1}, ОөвӮҖ = 0
Оө = [0, 0.50, вҲ’0.30, 0.80, вҲ’0.20, 0.60]

yвӮҒ = 0.50 + 0.5 Г— 0.00 = 0.5000
yвӮӮ = вҲ’0.30 + 0.5 Г— 0.50 = вҲ’0.30 + 0.25 = вҲ’0.0500
yвӮғ = 0.80 + 0.5 Г— (вҲ’0.30) = 0.80 вҲ’ 0.15 = 0.6500
yвӮ„ = вҲ’0.20 + 0.5 Г— 0.80 = вҲ’0.20 + 0.40 = 0.2000
yвӮ… = 0.60 + 0.5 Г— (вҲ’0.20) = 0.60 вҲ’ 0.10 = 0.5000
AR vs MA вҖ” Intuitive Difference
AR: "Today's value depends on yesterday's actual values." (infinite memory, decays)
MA: "Today's value depends on yesterday's shock/error." (finite memory, exactly q lags)

в‘ў ARIMA(1,1,1) вҖ” Full Worked Example

Series: y = [100, 102, 106, 111, 117, 122]
ADF test showed non-stationary вҶ’ d=1. ACF/PACF on О”y suggest p=1, q=1.
Step 1 вҖ” Difference (d=1): О”y = [102вҲ’100, 106вҲ’102, 111вҲ’106, 117вҲ’111, 122вҲ’117] = [2, 4, 5, 6, 5] Step 2 вҖ” ARMA(1,1) model on О”y_t: О”y_t = c + ПҶвӮҒВ·О”y_{tвҲ’1} + Оө_t + ОёвӮҒВ·Оө_{tвҲ’1} Estimated coefficients (from MLE): c = 0.5, ПҶвӮҒ = 0.60, ОёвӮҒ = 0.30 Step 3 вҖ” Forecast О”yвӮҶ: ОөвӮ… = О”yвӮ… вҲ’ (c + ПҶвӮҒВ·О”yвӮ„) = 5 вҲ’ (0.5 + 0.60Г—6) = 5 вҲ’ 4.1 = 0.90 О”Е·вӮҶ = 0.5 + 0.60Г—5 + 0.30Г—0.90 = 0.5 + 3.0 + 0.27 = 3.77 Step 4 вҖ” Undo differencing (integrate back): Е·вӮҶ = yвӮ… + О”Е·вӮҶ = 122 + 3.77 = 125.77

Model Selection Workflow

1. Plot series вҶ’ check for trend / seasonality / variance changes 2. ADF test вҶ’ if non-stationary, difference вҶ’ repeat ADF вҶ’ get d 3. Plot ACF + PACF on stationary series: вҖў PACF cuts off at p вҶ’ AR(p) вҖў ACF cuts off at q вҶ’ MA(q) вҖў Both tail off вҶ’ ARMA(p,q) 4. Fit candidate models: ARIMA(1,1,0), (0,1,1), (1,1,1), (2,1,0)... 5. Compare AIC / BIC (lower = better): AIC = 2k вҲ’ 2В·ln(L) BIC = kВ·ln(n) вҲ’ 2В·ln(L) вҶҗ stronger penalty for complexity 6. Check residuals: вҖў ACF of residuals вҶ’ should look like white noise (no pattern) вҖў Ljung-Box test: HвӮҖ = white noise вҶ’ want p-value > 0.05 вҖў Q-Q plot вҶ’ check normality 7. Forecast accuracy on holdout set: MAE, RMSE, MAPE

Python вҖ” Complete ARIMA Pipeline

import pandas as pd import numpy as np from statsmodels.tsa.arima.model import ARIMA from statsmodels.tsa.stattools import adfuller from statsmodels.graphics.tsaplots import plot_acf, plot_pacf from statsmodels.stats.diagnostic import acorr_ljungbox import matplotlib.pyplot as plt # 1. Load data ts = pd.Series([100, 102, 106, 111, 117, 122, 128, 134, 140, 145]) # 2. Test stationarity result = adfuller(ts, autolag='AIC') print(f"ADF p-value: {result[1]:.4f}") # > 0.05 вҶ’ non-stationary вҶ’ d=1 # 3. Difference dts = ts.diff().dropna() print(f"After diff, ADF p-value: {adfuller(dts)[1]:.4f}") # should be < 0.05 # 4. Plot ACF and PACF to choose p, q fig, axes = plt.subplots(1, 2, figsize=(12, 4)) plot_acf(dts, ax=axes[0], lags=8) # look for cutoff вҶ’ q plot_pacf(dts, ax=axes[1], lags=8) # look for cutoff вҶ’ p plt.show() # 5. Fit ARIMA(1,1,1) model = ARIMA(ts, order=(1, 1, 1)) result = model.fit() print(result.summary()) # 6. Forecast 3 steps ahead forecast = result.forecast(steps=3) pred_obj = result.get_forecast(steps=3) conf_int = pred_obj.conf_int(alpha=0.05) print("Forecasts:", forecast.values) print("95% CI:\n", conf_int) # 7. Residual diagnostics residuals = result.resid lb_test = acorr_ljungbox(residuals, lags=[10], return_df=True) print("Ljung-Box p-value:", lb_test['lb_pvalue'].values[0]) # p > 0.05 вҶ’ residuals look like white noise вң“ # 8. Auto-select p, d, q from pmdarima import auto_arima auto_model = auto_arima(ts, seasonal=False, information_criterion='aic', stepwise=True, suppress_warnings=True) print(auto_model.summary())

вҡҷпёҸ ARIMAx вҖ” ARIMA with eXogenous Variables

ARIMAX extends ARIMA by adding external predictor variables (X) that help explain the time series beyond its own past values. Think of it as ARIMA + regression.

ARIMAX(p, d, q) equation: О”бөҲy_t = c + ПҶвӮҒО”бөҲy_{tвҲ’1} + ... + ПҶвӮҡО”бөҲy_{tвҲ’p} + ОёвӮҒОө_{tвҲ’1} + ... + Оё_qОө_{tвҲ’q} + ОІвӮҒxвӮҒ_t + ОІвӮӮxвӮӮ_t + ... + ОІ_m x_m_t + Оө_t Where: x_{i,t} = exogenous variable i at time t ОІ_i = regression coefficient for x_i ARIMA part models the residual temporal structure Exogenous X is NOT differenced by default in statsmodels

вҡҷпёҸ SARIMA & SARIMAX вҖ” Seasonal Models

Notation: (p,d,q)(P,D,Q)[s]

(p, d, q) = non-seasonal part (same as ARIMA) (P, D, Q) = seasonal part [s] = seasonal period Common seasonal periods: s = 12 вҶ’ monthly data (annual seasonality) s = 4 вҶ’ quarterly data s = 7 вҶ’ daily data (weekly seasonality) s = 24 вҶ’ hourly data (daily seasonality) Full SARIMA using backshift operator B: ОҰ_P(B^s) В· ПҶ_p(B) В· (1вҲ’B^s)^D В· (1вҲ’B)^d В· y_t = Оҳ_Q(B^s) В· Оё_q(B) В· Оө_t Where: ПҶ_p(B) = 1 вҲ’ ПҶвӮҒB вҲ’ ... вҲ’ ПҶвӮҡB^p (non-seasonal AR) ОҰ_P(B^s) = 1 вҲ’ ОҰвӮҒB^s вҲ’ ... вҲ’ ОҰ_PB^{Ps} (seasonal AR) Оё_q(B) = 1 + ОёвӮҒB + ... + Оё_qB^q (non-seasonal MA) Оҳ_Q(B^s) = 1 + ОҳвӮҒB^s + ... + Оҳ_QB^{Qs} (seasonal MA)

рҹ”ў SARIMA(1,1,1)(1,1,1)[12] вҖ” Worked Example

Setup: Monthly retail sales with annual seasonality (s=12).
Non-seasonal trend + seasonal peak each December.
Full equation of SARIMA(1,1,1)(1,1,1)[12]: (1 вҲ’ ОҰвӮҒBВ№ВІ)(1 вҲ’ ПҶвӮҒB)(1вҲ’BВ№ВІ)(1вҲ’B) y_t = (1 + ОҳвӮҒBВ№ВІ)(1 + ОёвӮҒB) Оө_t Expanding the differencing: Non-seasonal diff: О”y_t = y_t вҲ’ y_{tвҲ’1} Seasonal diff: О”вӮҒвӮӮy_t = y_t вҲ’ y_{tвҲ’12} Both combined: О”В·О”вӮҒвӮӮy_t = y_t вҲ’ y_{tвҲ’1} вҲ’ y_{tвҲ’12} + y_{tвҲ’13} Estimated parameters (example): ПҶвӮҒ = 0.40 (non-seasonal AR: recent momentum) ОҰвӮҒ = 0.65 (seasonal AR: same month last year) ОёвӮҒ = 0.30 (non-seasonal MA: recent shock) ОҳвӮҒ = 0.50 (seasonal MA: seasonal shock) Forecast for month 25 (uses yвӮӮвӮ„, yвӮӮвӮғ, yвӮҒвӮғ, yвӮҒвӮӮ + residuals)

Seasonal Differencing Rule: D=1 is almost always enough. After one seasonal difference, check if ACF spike at lag s disappears. D=2 is very rare. Over-differencing (D+d > 2) usually indicates a problem.

рҹҺӨ Interview Q&A вҖ” 10 Must-Know Questions

Click any question to reveal the model answer.
Q1What is the difference between ACF and PACF? When do you use each?Fundamentalsв–ј
ACF (AutoCorrelation Function) measures the correlation between y_t and y_{tвҲ’k} including ALL intermediate lag effects. PACF measures the DIRECT correlation after removing all effects from lags 1 through kвҲ’1.

Use ACF to determine q (MA order) вҖ” a sharp cutoff at lag q means MA(q). Use PACF to determine p (AR order) вҖ” a sharp cutoff at lag p means AR(p).

Memory hook: AR вҶ’ PACF cuts off вҶ’ use PACF to find p. MA вҶ’ ACF cuts off вҶ’ use ACF to find q. If both tail off, you need ARMA(p,q).
Q2What is a unit root and why does it matter?Core Conceptв–ј
A unit root means ПҶ=1 in y_t = ПҶВ·y_{tвҲ’1} + Оө_t, turning the series into a random walk. Shocks NEVER die out; the series has infinite memory and no tendency to revert to the mean.

It matters because OLS on non-stationary series gives spurious regression вҖ” high RВІ, significant t-stats, but the relationship is completely meaningless (both series are just trending). We test for unit roots using the ADF test and fix them by differencing the series.
Q3Explain the difference between d and D in SARIMA(p,d,q)(P,D,Q)[s].SARIMAв–ј
d = non-seasonal differencing: removes trend. О”y_t = y_t вҲ’ y_{tвҲ’1}.
D = seasonal differencing: removes seasonal trend. О”_s y_t = y_t вҲ’ y_{tвҲ’s}.

Applied together: О”В·О”_s y_t = y_t вҲ’ y_{tвҲ’1} вҲ’ y_{tвҲ’s} + y_{tвҲ’sвҲ’1}.

d handles slowly trending behavior; D handles persistent seasonal cycles. Usually d вүӨ 2, D = 1. Over-differencing (d+D > 2) is a red flag вҖ” simplify the model.
Q4ADF test p-value is 0.30. What do you do next?ADF Testв–ј
p-value 0.30 > 0.05 вҶ’ fail to reject HвӮҖ (unit root present) вҶ’ series is NON-STATIONARY.

Steps:
1. Apply 1st differencing: О”y_t = y_t вҲ’ y_{tвҲ’1}
2. Run ADF again on О”y_t
3. If p < 0.05 вҶ’ confirmed d=1, proceed
4. If still failing вҶ’ try d=2 (rare)

Also run KPSS as a cross-check. If ADF fails + KPSS also rejects stationarity, you're very confident it's non-stationary. Also plot the series visually вҖ” sometimes a structural break fools ADF.
Q5What is the difference between ARIMA and ARIMAx?ARIMAxв–ј
ARIMA models the series using only its own past values and past errors. ARIMAx adds exogenous variables X_t вҖ” external predictors (e.g., temperature, GDP, promotional spend) вҖ” that can explain variation beyond what's in the series itself.

Key requirement: future X values must be available at forecast time. If forecasting next month's sales, you need next month's temperature first.

The ARIMA part accounts for residual autocorrelation that simple linear regression ignores вҖ” this is the key advantage over just adding regressors to OLS.
Q6How do you choose between AIC and BIC for model selection?Model Selectionв–ј
AIC = 2k вҲ’ 2В·ln(L). BIC = kВ·ln(n) вҲ’ 2В·ln(L). Both reward fit and penalize parameters k.

BIC has a STRONGER penalty (kВ·ln(n) vs 2k when n > 7), so it prefers simpler models.

Use AIC when forecast accuracy is the goal вҖ” it tends to keep more parameters which helps prediction.
Use BIC when interpretability / parsimony matters вҖ” it finds the simplest model that describes the data.

In practice: fit several candidates, compare AIC/BIC, then validate all finalists on a holdout set with RMSE or MAPE to make the final call.
Q7Residuals from ARIMA show significant ACF at lag 12. What does this tell you and what do you do?Diagnosticsв–ј
The residuals are NOT white noise вҖ” there's unmodeled seasonal structure at period 12. The ARIMA model is missing a seasonal component.

Solution: switch to SARIMA(p,d,q)(P,D,Q)[12]. Start with (1,1,1)(1,1,1)[12] and refine P, Q using seasonal ACF/PACF patterns.

General residual checklist:
вҖў ACF/PACF of residuals вҶ’ all within CI band (white noise)
вҖў Ljung-Box test: HвӮҖ = white noise вҶ’ want p-value > 0.05
вҖў Q-Q plot вҶ’ normality
вҖў Plot residuals over time вҶ’ no patterns or heteroscedasticity
Q8Explain invertibility of MA models вҖ” why does it matter?Theoryв–ј
An MA(q) model is invertible if it can be written as an infinite AR process вҖ” which requires all roots of the MA polynomial Оё(B)=0 to lie outside the unit circle.

Invertibility ensures a unique MA representation. Without it, multiple parameter values produce the same ACF, making estimation ill-defined. For MA(1): y_t = Оө_t + ОёОө_{tвҲ’1} вҖ” models with Оё and 1/Оё are observationally equivalent. By convention we constrain |Оё| < 1 to get the unique invertible form.

statsmodels enforces invertibility by default via the enforce_invertibility=True parameter.
Q9When would you NOT use ARIMA? What are its key limitations?Limitationsв–ј
ARIMA fails when:
вҖў Non-linear patterns exist вҶ’ use neural networks, XGBoost on time features, or NHiTS
вҖў Heteroscedasticity (changing variance) вҶ’ use GARCH/EGARCH (finance)
вҖў Multiple seasonalities (e.g. hourly data: daily + weekly pattern) вҶ’ use Prophet or TBATS
вҖў Structural breaks вҶ’ ARIMA assumes fixed parameters over time
вҖў Multiple interacting series вҶ’ use VAR (Vector AutoRegression)
вҖў Very long-range dependencies вҶ’ use ARFIMA or deep learning
вҖў Extreme events / fat tails вҶ’ ARIMA's Gaussian errors underestimate risk
Q10What is spurious regression? How do you detect and fix it?Regression Trapsв–ј
When two non-stationary series are regressed against each other, you get high RВІ, significant t-stats вҖ” but the relationship is completely meaningless because both series are simply trending together.

Classic example: US divorce rate vs margarine consumption. RВІ вүҲ 0.99. Completely spurious.

Detection: Run ADF on both series вҖ” if both are I(1), suspect spurious regression. Also: very high RВІ but low Durbin-Watson statistic (DW < 1) is a red flag.

Fix Option 1: Difference both series before regression (lose long-run info).
Fix Option 2: Test for cointegration (Engle-Granger or Johansen test). If cointegrated, a genuine long-run relationship exists вҶ’ use an Error Correction Model (ECM) which captures both long-run equilibrium and short-run dynamics.
