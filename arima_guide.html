TIME SERIES FORECASTING MASTERCLASS

ARIMA · ARIMAx · SARIMAx · ACF · PACF · ADF TEST — INTERVIEW READY

📐 Stationarity — The Foundation of Everything

A time series is stationary if its statistical properties (mean, variance, autocorrelation) do not change over time. Almost every classical forecasting model requires stationarity.

Formal Definition (Weak / Covariance Stationarity)
A series Yt is weakly stationary if:
1. E[Yt] = μ — constant mean
2. Var(Yt) = σ² — constant variance
3. Cov(Yt, Yt−k) = γ(k) — depends only on lag k, not on t

Types of Non-Stationarity

TypeDescriptionFix
TrendMean grows/shrinks over timeDifferencing (d ≥ 1)
HeteroscedasticityVariance changes over timeLog transform, Box-Cox
SeasonalityPeriodic patterns repeatSeasonal differencing (D ≥ 1)
Structural BreakSudden regime changeDummy variables, regime models
Unit RootStochastic trend (random walk)Differencing d times

🔢 Differencing — The Math

1st Difference: Δy_t = y_t − y_{t−1} 2nd Difference: Δ²y_t = Δy_t − Δy_{t−1} = y_t − 2y_{t−1} + y_{t−2} Seasonal Diff (period s): Δ_s y_t = y_t − y_{t−s} Backshift Operator B: B·y_t = y_{t−1} → Δy_t = (1−B)y_t → Δ^d y_t = (1−B)^d y_t
Numerical Example
y = [100, 102, 106, 111, 117, 122]

1st Diff (Δy): 102−100=2, 106−102=4, 111−106=5, 117−111=6, 122−117=5
2nd Diff (Δ²y): 4−2=2, 5−4=1, 6−5=1, 5−6=−1

The 2nd difference removes the accelerating trend. Series now fluctuates around 0.

Visual: Non-Stationary vs Stationary After Differencing

NON-STATIONARY: y=[100,102,106,111,117,122,128,134,140,145]
AFTER 1st DIFF: Δy=[2,4,5,6,5,6,6,6,5]

How Many Times to Difference? (Choosing d)

Run the ADF test (Tab ④) after each differencing. If p-value > 0.05, difference again. Usually d = 0, 1, or 2. Over-differencing introduces unnecessary MA terms — stop as soon as ADF confirms stationarity.

⚠️ Over-differencing Warning
If you difference more than needed, you artificially introduce autocorrelation into the series. d=2 should only be used if d=1 still fails the ADF test. Always verify visually too.

📊 AutoCorrelation Function (ACF)

The ACF measures the linear correlation between a time series and its own lagged version. It includes both direct and indirect effects of all intermediate lags.

Σ_i (y_i − ȳ)(y_{i−k} − ȳ) ρ(k) = ───────────────────────────── Σ_i (y_i − ȳ)² Where: k = lag, ȳ = sample mean ρ(0) = 1 always ρ(k) ∈ [−1, 1]

📊 Partial AutoCorrelation Function (PACF)

PACF measures the correlation between y_t and y_{t−k} after removing the effect of all intermediate lags (1 through k−1). It isolates the direct relationship of lag k only.

PACF(k) = Corr(y_t, y_{t−k} | y_{t−1}, y_{t−2}, ..., y_{t−k+1}) Computed via Yule-Walker equations (Levinson-Durbin recursion): PACF(1) = φ₁₁ = ρ(1) [no intermediate lags to remove] PACF(2) = φ₂₂ = [ρ(2) − ρ(1)²] / [1 − ρ(1)²] For lag k: regress y_t on y_{t−1},...,y_{t−k} φ_kk = coefficient on y_{t−k} (last coefficient)

🔢 Step-by-Step Calculation

Same series: y = [3,5,4,6,5,7,6,8,7,9]  |  ρ(1) = 0.3000  |  ρ(2) = 0.5333
PACF(1) = ρ(1) = 0.3000 (Same as ACF at lag 1 — no intermediates to control for) PACF(2) = [ρ(2) − ρ(1)²] / [1 − ρ(1)²] = [0.5333 − 0.3000²] / [1 − 0.3000²] = [0.5333 − 0.0900] / [1 − 0.0900] = 0.4433 / 0.9100 ≈ 0.4871 Interpretation: After removing the lag-1 effect, lag-2 still has a direct correlation of 0.487 with the current value.

95% Confidence Interval on ACF/PACF Plots

Bartlett's approximation: ± 1.96 / √n n = 10 → ±0.620 (wide — small sample, unreliable) n = 100 → ±0.196 n = 500 → ±0.088 (tight — small effects detectable) Any bar extending BEYOND the dashed band is statistically significant at the 5% level. Larger samples → narrower bands → more reliable plots.

🧪 Augmented Dickey-Fuller (ADF) Test

The ADF test formally checks whether a unit root exists in a time series. A unit root = non-stationary (series behaves like a random walk).

Hypotheses:
H₀ (Null): Unit root exists → series is NON-stationary
H₁ (Alternative): No unit root → series IS stationary

p-value < 0.05 → Reject H₀ → STATIONARY ✓
p-value ≥ 0.05 → Fail to reject H₀ → NON-STATIONARY → difference!

The Math — Where Does the Test Come From?

Start with AR(1): y_t = φ·y_{t−1} + ε_t Subtract y_{t−1}: Δy_t = (φ−1)·y_{t−1} + ε_t Let δ = φ − 1: Δy_t = δ·y_{t−1} + ε_t H₀: δ = 0 (φ=1 → unit root → non-stationary → Δy_t = ε_t, pure noise) H₁: δ < 0 (φ<1 → stationary → series mean-reverts) AUGMENTED version — adds lagged differences to remove autocorrelation in residuals (p lags chosen via AIC/BIC): Δy_t = α + β·t + δ·y_{t−1} + Σⱼ γⱼ·Δy_{t−j} + ε_t ↑ ↑ ↑ const trend ← this is the test term Test statistic: τ = δ̂ / SE(δ̂) (NOT standard t-distribution!)

3 Variants — Which to Use?

VariantEquation IncludesUse When
No constant, no trendonly δ·y_{t−1}Series fluctuates around 0
Constant, no trendα + δ·y_{t−1}Series has non-zero mean (most common)
Constant + trendα + βt + δ·y_{t−1}Series has a deterministic trend

🔢 Numerical Walkthrough

Random Walk (non-stationary): y_t = y_{t−1} + ε_t
y = [0, 0.5, 0.8, 0.3, 0.9, 1.4, 1.1, 1.7, 2.1, 1.8]
Δy = [0.5, 0.3, −0.5, 0.6, 0.5, −0.3, 0.6, 0.4, −0.3]

Regress Δy_t on y_{t−1}: expect δ ≈ 0 → t-stat near 0 → p-value > 0.05
→ Cannot reject H₀ → NON-STATIONARY → apply differencing
Stationary AR(1), φ=0.5: y_t = 0.5·y_{t−1} + ε_t
δ = φ − 1 = 0.5 − 1 = −0.5
This negative value is significantly different from 0 → t-stat is very negative
→ Compare to MacKinnon critical value (e.g. −2.86 at 5%)
→ t-stat < −2.86 → Reject H₀ → STATIONARY ✓

Python Code

from statsmodels.tsa.stattools import adfuller result = adfuller(y, autolag='AIC') # AIC selects lag length p print(f"ADF Statistic : {result[0]:.4f}") print(f"p-value : {result[1]:.4f}") print(f"Lags Used : {result[2]}") print(f"Critical Values:") for k, v in result[4].items(): print(f" {k}: {v:.3f}") # Interpret if result[1] < 0.05: print("→ STATIONARY (reject H₀)") else: print("→ NON-STATIONARY → apply differencing!") # Also run KPSS from statsmodels.tsa.stattools import kpss stat, p_val, lags, crit = kpss(y, regression='c') print(f"KPSS p-value: {p_val:.4f}") # KPSS: p < 0.05 → reject H₀ of stationarity → non-stationary

⚙️ ARIMA(p, d, q) — AutoRegressive Integrated Moving Average

ARIMA(p, d, q) = AR(p) + I(d) + MA(q) p = order of AutoRegressive part d = degree of differencing (makes series stationary) q = order of Moving Average part Full ARIMA equation on differenced series Δᵈy_t: Δᵈy_t = c + φ₁Δᵈy_{t−1} + ... + φₚΔᵈy_{t−p} + ε_t + θ₁ε_{t−1} + ... + θ_qε_{t−q} Using backshift operator B (B·y_t = y_{t−1}): φ(B)(1−B)ᵈ y_t = c + θ(B)ε_t where φ(B) = 1 − φ₁B − φ₂B² − ... − φₚBᵖ (AR polynomial) θ(B) = 1 + θ₁B + θ₂B² + ... + θ_qBᵍ (MA polynomial)

① AR(p) — AutoRegressive Model

y_t = c + φ₁y_{t−1} + φ₂y_{t−2} + ... + φₚy_{t−p} + ε_t ε_t ~ N(0, σ²) (white noise, IID) Stationarity condition: all roots of φ(B)=0 lie OUTSIDE the unit circle For AR(1): |φ₁| < 1
🔢 AR(1) Walkthrough: y_t = 0.7·y_{t−1} + ε_t, y₀ = 2
ε = [0.5, −0.3, 0.8, −0.2, 0.6]

y₁ = 0.7 × 2.0000 + 0.5000 = 1.9000
y₂ = 0.7 × 1.9000 + (−0.3000) = 1.3300 − 0.3000 = 1.0300
y₃ = 0.7 × 1.0300 + 0.8000 = 0.7210 + 0.8000 = 1.5210
y₄ = 0.7 × 1.5210 + (−0.2000) = 1.0647 − 0.2000 = 0.8647
y₅ = 0.7 × 0.8647 + 0.6000 = 0.6053 + 0.6000 = 1.2053
Key AR Properties:
|φ| < 1 → stationary (mean-reverts to μ/(1−φ₁))
φ = 1 → unit root → random walk (non-stationary, needs differencing)
|φ| > 1 → explosive (grows without bound)
Larger |φ| → stronger memory, slower geometric decay in ACF

② MA(q) — Moving Average Model

y_t = μ + ε_t + θ₁ε_{t−1} + θ₂ε_{t−2} + ... + θ_qε_{t−q} Always stationary regardless of θ values! Invertibility condition: all roots of θ(B)=0 lie OUTSIDE unit circle For MA(1): |θ₁| < 1 Memory: exactly q lags (finite memory)
🔢 MA(1) Walkthrough: y_t = ε_t + 0.5·ε_{t−1}, ε₀ = 0
ε = [0, 0.50, −0.30, 0.80, −0.20, 0.60]

y₁ = 0.50 + 0.5 × 0.00 = 0.5000
y₂ = −0.30 + 0.5 × 0.50 = −0.30 + 0.25 = −0.0500
y₃ = 0.80 + 0.5 × (−0.30) = 0.80 − 0.15 = 0.6500
y₄ = −0.20 + 0.5 × 0.80 = −0.20 + 0.40 = 0.2000
y₅ = 0.60 + 0.5 × (−0.20) = 0.60 − 0.10 = 0.5000
AR vs MA — Intuitive Difference
AR: "Today's value depends on yesterday's actual values." (infinite memory, decays)
MA: "Today's value depends on yesterday's shock/error." (finite memory, exactly q lags)

③ ARIMA(1,1,1) — Full Worked Example

Series: y = [100, 102, 106, 111, 117, 122]
ADF test showed non-stationary → d=1. ACF/PACF on Δy suggest p=1, q=1.
Step 1 — Difference (d=1): Δy = [102−100, 106−102, 111−106, 117−111, 122−117] = [2, 4, 5, 6, 5] Step 2 — ARMA(1,1) model on Δy_t: Δy_t = c + φ₁·Δy_{t−1} + ε_t + θ₁·ε_{t−1} Estimated coefficients (from MLE): c = 0.5, φ₁ = 0.60, θ₁ = 0.30 Step 3 — Forecast Δy₆: ε₅ = Δy₅ − (c + φ₁·Δy₄) = 5 − (0.5 + 0.60×6) = 5 − 4.1 = 0.90 Δŷ₆ = 0.5 + 0.60×5 + 0.30×0.90 = 0.5 + 3.0 + 0.27 = 3.77 Step 4 — Undo differencing (integrate back): ŷ₆ = y₅ + Δŷ₆ = 122 + 3.77 = 125.77

Model Selection Workflow

1. Plot series → check for trend / seasonality / variance changes 2. ADF test → if non-stationary, difference → repeat ADF → get d 3. Plot ACF + PACF on stationary series: • PACF cuts off at p → AR(p) • ACF cuts off at q → MA(q) • Both tail off → ARMA(p,q) 4. Fit candidate models: ARIMA(1,1,0), (0,1,1), (1,1,1), (2,1,0)... 5. Compare AIC / BIC (lower = better): AIC = 2k − 2·ln(L) BIC = k·ln(n) − 2·ln(L) ← stronger penalty for complexity 6. Check residuals: • ACF of residuals → should look like white noise (no pattern) • Ljung-Box test: H₀ = white noise → want p-value > 0.05 • Q-Q plot → check normality 7. Forecast accuracy on holdout set: MAE, RMSE, MAPE

Python — Complete ARIMA Pipeline

import pandas as pd import numpy as np from statsmodels.tsa.arima.model import ARIMA from statsmodels.tsa.stattools import adfuller from statsmodels.graphics.tsaplots import plot_acf, plot_pacf from statsmodels.stats.diagnostic import acorr_ljungbox import matplotlib.pyplot as plt # 1. Load data ts = pd.Series([100, 102, 106, 111, 117, 122, 128, 134, 140, 145]) # 2. Test stationarity result = adfuller(ts, autolag='AIC') print(f"ADF p-value: {result[1]:.4f}") # > 0.05 → non-stationary → d=1 # 3. Difference dts = ts.diff().dropna() print(f"After diff, ADF p-value: {adfuller(dts)[1]:.4f}") # should be < 0.05 # 4. Plot ACF and PACF to choose p, q fig, axes = plt.subplots(1, 2, figsize=(12, 4)) plot_acf(dts, ax=axes[0], lags=8) # look for cutoff → q plot_pacf(dts, ax=axes[1], lags=8) # look for cutoff → p plt.show() # 5. Fit ARIMA(1,1,1) model = ARIMA(ts, order=(1, 1, 1)) result = model.fit() print(result.summary()) # 6. Forecast 3 steps ahead forecast = result.forecast(steps=3) pred_obj = result.get_forecast(steps=3) conf_int = pred_obj.conf_int(alpha=0.05) print("Forecasts:", forecast.values) print("95% CI:\n", conf_int) # 7. Residual diagnostics residuals = result.resid lb_test = acorr_ljungbox(residuals, lags=[10], return_df=True) print("Ljung-Box p-value:", lb_test['lb_pvalue'].values[0]) # p > 0.05 → residuals look like white noise ✓ # 8. Auto-select p, d, q from pmdarima import auto_arima auto_model = auto_arima(ts, seasonal=False, information_criterion='aic', stepwise=True, suppress_warnings=True) print(auto_model.summary())

⚙️ ARIMAx — ARIMA with eXogenous Variables

ARIMAX extends ARIMA by adding external predictor variables (X) that help explain the time series beyond its own past values. Think of it as ARIMA + regression.

ARIMAX(p, d, q) equation: Δᵈy_t = c + φ₁Δᵈy_{t−1} + ... + φₚΔᵈy_{t−p} + θ₁ε_{t−1} + ... + θ_qε_{t−q} + β₁x₁_t + β₂x₂_t + ... + β_m x_m_t + ε_t Where: x_{i,t} = exogenous variable i at time t β_i = regression coefficient for x_i ARIMA part models the residual temporal structure Exogenous X is NOT differenced by default in statsmodels

⚠️ Critical ARIMAX Requirement
You MUST have the future values of X available at forecast time!
If forecasting next month's sales, you need to know (or forecast) next month's temperature first.
If X is unknown → use lagged X (x_{t−1}, x_{t−2}) or model X separately.

⭐ Naming Confusion — Know This for Interviews

statsmodels ARIMA(exog=X) = "Regression with ARIMA errors"
→ X is NOT differenced; only the ARIMA errors are modelled

True ARIMAX = X is also differenced with the series

Transfer Function Model = X can have its own lag structure (e.g. β₀x_t + β₁x_{t−1})

In practice: use SARIMAX from statsmodels — it handles all these cases

Python Code

from statsmodels.tsa.statespace.sarimax import SARIMAX import numpy as np y = np.array([100, 112, 130, 145, 155, 162]) X = np.array([15, 18, 22, 26, 28, 30]).reshape(-1, 1) # exogenous: temp # Fit ARIMAX(1,1,0) using SARIMAX (recommended for all ARIMA-family models) model = SARIMAX(y, exog = X, order = (1, 1, 0), trend = 'c', enforce_stationarity = True, enforce_invertibility = True) result = model.fit(disp=False) print(result.summary()) # Forecast month 7 — MUST provide future X value! X_future = np.array([[29]]) # temperature for month 7 forecast = result.forecast(steps=1, exog=X_future) print(f"Month 7 forecast: {forecast[0]:.2f}") # Confidence interval pred = result.get_forecast(steps=1, exog=X_future) ci = pred.conf_int(alpha=0.05) print(f"95% CI: [{ci.iloc[0,0]:.2f}, {ci.iloc[0,1]:.2f}]") # Multiple exogenous variables X2 = np.column_stack([temp, humidity, holiday_dummy]) model2 = SARIMAX(y, exog=X2, order=(1,1,1)).fit(disp=False)

⚙️ SARIMA & SARIMAX — Seasonal Models

Notation: (p,d,q)(P,D,Q)[s]

(p, d, q) = non-seasonal part (same as ARIMA) (P, D, Q) = seasonal part [s] = seasonal period Common seasonal periods: s = 12 → monthly data (annual seasonality) s = 4 → quarterly data s = 7 → daily data (weekly seasonality) s = 24 → hourly data (daily seasonality) Full SARIMA using backshift operator B: Φ_P(B^s) · φ_p(B) · (1−B^s)^D · (1−B)^d · y_t = Θ_Q(B^s) · θ_q(B) · ε_t Where: φ_p(B) = 1 − φ₁B − ... − φₚB^p (non-seasonal AR) Φ_P(B^s) = 1 − Φ₁B^s − ... − Φ_PB^{Ps} (seasonal AR) θ_q(B) = 1 + θ₁B + ... + θ_qB^q (non-seasonal MA) Θ_Q(B^s) = 1 + Θ₁B^s + ... + Θ_QB^{Qs} (seasonal MA)

🔢 SARIMA(1,1,1)(1,1,1)[12] — Worked Example

Setup: Monthly retail sales with annual seasonality (s=12).
Non-seasonal trend + seasonal peak each December.
Full equation of SARIMA(1,1,1)(1,1,1)[12]: (1 − Φ₁B¹²)(1 − φ₁B)(1−B¹²)(1−B) y_t = (1 + Θ₁B¹²)(1 + θ₁B) ε_t Expanding the differencing: Non-seasonal diff: Δy_t = y_t − y_{t−1} Seasonal diff: Δ₁₂y_t = y_t − y_{t−12} Both combined: Δ·Δ₁₂y_t = y_t − y_{t−1} − y_{t−12} + y_{t−13} Estimated parameters (example): φ₁ = 0.40 (non-seasonal AR: recent momentum) Φ₁ = 0.65 (seasonal AR: same month last year) θ₁ = 0.30 (non-seasonal MA: recent shock) Θ₁ = 0.50 (seasonal MA: seasonal shock) Forecast for month 25 (uses y₂₄, y₂₃, y₁₃, y₁₂ + residuals)

PACFCuts off after lag sSeasonal AR(1) → P=1
PACFSlow decay at s,2s,3sSeasonal MA needed → Q≥1
BothTailing off at seasonal lagsSeasonal ARMA → P≥1, Q≥1
Seasonal Differencing Rule: D=1 is almost always enough. After one seasonal difference, check if ACF spike at lag s disappears. D=2 is very rare. Over-differencing (D+d > 2) usually indicates a problem.

SARIMAX = SARIMA + Exogenous Variables

SARIMAX(p,d,q)(P,D,Q)[s] with exogenous X: Φ(B^s)·φ(B)·(1−B^s)^D·(1−B)^d · y_t = β·X_t + Θ(B^s)·θ(B)·ε_t Real-world examples: • Energy demand (s=24 hourly): X = [temperature, day_of_week, holiday] • Retail sales (s=12 monthly): X = [promotions, GDP_index, holiday_dummy] • Airline pax (s=12 monthly): X = [ticket_price, fuel_cost, GDP] • Hospital admissions: X = [vaccination_rate, lockdown_dummy]

🎤 Interview Q&A — 10 Must-Know Questions

Click any question to reveal the model answer.
Q1What is the difference between ACF and PACF? When do you use each?Fundamentals▼
ACF (AutoCorrelation Function) measures the correlation between y_t and y_{t−k} including ALL intermediate lag effects. PACF measures the DIRECT correlation after removing all effects from lags 1 through k−1.

Use ACF to determine q (MA order) — a sharp cutoff at lag q means MA(q). Use PACF to determine p (AR order) — a sharp cutoff at lag p means AR(p).

Memory hook: AR → PACF cuts off → use PACF to find p. MA → ACF cuts off → use ACF to find q. If both tail off, you need ARMA(p,q).
Q2What is a unit root and why does it matter?Core Concept▼
A unit root means φ=1 in y_t = φ·y_{t−1} + ε_t, turning the series into a random walk. Shocks NEVER die out; the series has infinite memory and no tendency to revert to the mean.

It matters because OLS on non-stationary series gives spurious regression — high R², significant t-stats, but the relationship is completely meaningless (both series are just trending). We test for unit roots using the ADF test and fix them by differencing the series.
Q3Explain the difference between d and D in SARIMA(p,d,q)(P,D,Q)[s].SARIMA▼
d = non-seasonal differencing: removes trend. Δy_t = y_t − y_{t−1}.
D = seasonal differencing: removes seasonal trend. Δ_s y_t = y_t − y_{t−s}.

Applied together: Δ·Δ_s y_t = y_t − y_{t−1} − y_{t−s} + y_{t−s−1}.

d handles slowly trending behavior; D handles persistent seasonal cycles. Usually d ≤ 2, D = 1. Over-differencing (d+D > 2) is a red flag — simplify the model.
Q4ADF test p-value is 0.30. What do you do next?ADF Test▼
p-value 0.30 > 0.05 → fail to reject H₀ (unit root present) → series is NON-STATIONARY.

Steps:
1. Apply 1st differencing: Δy_t = y_t − y_{t−1}
2. Run ADF again on Δy_t
3. If p < 0.05 → confirmed d=1, proceed
4. If still failing → try d=2 (rare)

Also run KPSS as a cross-check. If ADF fails + KPSS also rejects stationarity, you're very confident it's non-stationary. Also plot the series visually — sometimes a structural break fools ADF.
Q5What is the difference between ARIMA and ARIMAx?ARIMAx▼
ARIMA models the series using only its own past values and past errors. ARIMAx adds exogenous variables X_t — external predictors (e.g., temperature, GDP, promotional spend) — that can explain variation beyond what's in the series itself.

Key requirement: future X values must be available at forecast time. If forecasting next month's sales, you need next month's temperature first.

The ARIMA part accounts for residual autocorrelation that simple linear regression ignores — this is the key advantage over just adding regressors to OLS.
Q6How do you choose between AIC and BIC for model selection?Model Selection▼
AIC = 2k − 2·ln(L). BIC = k·ln(n) − 2·ln(L). Both reward fit and penalize parameters k.

BIC has a STRONGER penalty (k·ln(n) vs 2k when n > 7), so it prefers simpler models.

Use AIC when forecast accuracy is the goal — it tends to keep more parameters which helps prediction.
Use BIC when interpretability / parsimony matters — it finds the simplest model that describes the data.

In practice: fit several candidates, compare AIC/BIC, then validate all finalists on a holdout set with RMSE or MAPE to make the final call.
Q7Residuals from ARIMA show significant ACF at lag 12. What does this tell you and what do you do?Diagnostics▼
The residuals are NOT white noise — there's unmodeled seasonal structure at period 12. The ARIMA model is missing a seasonal component.

Solution: switch to SARIMA(p,d,q)(P,D,Q)[12]. Start with (1,1,1)(1,1,1)[12] and refine P, Q using seasonal ACF/PACF patterns.

General residual checklist:
• ACF/PACF of residuals → all within CI band (white noise)
• Ljung-Box test: H₀ = white noise → want p-value > 0.05
• Q-Q plot → normality
• Plot residuals over time → no patterns or heteroscedasticity
Q8Explain invertibility of MA models — why does it matter?Theory▼
An MA(q) model is invertible if it can be written as an infinite AR process — which requires all roots of the MA polynomial θ(B)=0 to lie outside the unit circle.

Invertibility ensures a unique MA representation. Without it, multiple parameter values produce the same ACF, making estimation ill-defined. For MA(1): y_t = ε_t + θε_{t−1} — models with θ and 1/θ are observationally equivalent. By convention we constrain |θ| < 1 to get the unique invertible form.

statsmodels enforces invertibility by default via the enforce_invertibility=True parameter.
Q9When would you NOT use ARIMA? What are its key limitations?Limitations▼
ARIMA fails when:
• Non-linear patterns exist → use neural networks, XGBoost on time features, or NHiTS
• Heteroscedasticity (changing variance) → use GARCH/EGARCH (finance)
• Multiple seasonalities (e.g. hourly data: daily + weekly pattern) → use Prophet or TBATS
• Structural breaks → ARIMA assumes fixed parameters over time
• Multiple interacting series → use VAR (Vector AutoRegression)
• Very long-range dependencies → use ARFIMA or deep learning
• Extreme events / fat tails → ARIMA's Gaussian errors underestimate risk
Q10What is spurious regression? How do you detect and fix it?Regression Traps▼
When two non-stationary series are regressed against each other, you get high R², significant t-stats — but the relationship is completely meaningless because both series are simply trending together.

Classic example: US divorce rate vs margarine consumption. R² ≈ 0.99. Completely spurious.

Detection: Run ADF on both series — if both are I(1), suspect spurious regression. Also: very high R² but low Durbin-Watson statistic (DW < 1) is a red flag.

Fix Option 1: Difference both series before regression (lose long-run info).
Fix Option 2: Test for cointegration (Engle-Granger or Johansen test). If cointegrated, a genuine long-run relationship exists → use an Error Correction Model (ECM) which captures both long-run equilibrium and short-run dynamics.
