String Methods — The .str Accessor

The core rule: any Python string method (.split, .contains, .lower, .replace, .strip) does NOT work directly on a pandas column. A column (Series) is a container of strings, not a string itself — you must go through the .str accessor to apply the method to every row at once. This is one of the most common sources of syntax errors under time pressure, so it's worth internalizing early.

# WRONG — Series has no .split() method df['category_path'].split('/') # RIGHT — .str broadcasts the string method across every row df['category_path'].str.split('/')

Most-used .str methods

MethodWhat it doesExample
.str.split('/')Splits each string into a list"a/b/c" → ['a','b','c']
.str.split('/').str[0]Split then grab one element (chain .str twice)"a/b/c" → "a"
.str.contains('phone', case=False, na=False)Boolean mask — substring match, regex by default"iPhone 13" → True
.str.lower() / .str.upper()Case conversion per row"Nike" → "nike"
.str.replace(r'\D', '', regex=True)Regex-based find/replace"(123) 456-7890" → "1234567890"
.str.strip()Removes leading/trailing whitespace" abc " → "abc"
.str.get('key')Dict-style key access when column holds dicts{'brand':'Nike'} → "Nike"
.str.len()String length per row"abc" → 3
💡

Matching against a list of keywords is a common trap. .str.contains() takes a single pattern, not a list. To check for "any of these words," join the list with regex OR: pattern = '|'.join(['diamond','gold','luxury']) then .str.contains(pattern, case=False, na=False).

Quick Check
Q. Why does na=False matter in .str.contains()?
A. If a row is null, .str.contains() returns NaN by default instead of True/False — that breaks boolean indexing (df[mask] errors or misbehaves on NaN). na=False treats nulls as "no match."
Q. Why use case=False instead of chaining .str.lower().str.contains()?
A. Fewer operations, same result — case=False is a built-in parameter, so there's no need to transform the whole column first.
Interview Trap

Chaining .str.lower().contains(...) without the second .str — every .str. method call returns a plain Series again, so the *next* string method also needs its own .str. prefix. There's no auto-chaining.

Date & Time Handling

df['order_date'] = pd.to_datetime(df['order_date'], errors='coerce') # NOT coerce_errors= today = pd.Timestamp.now() # no import needed beyond pandas df['days_since_order'] = (today - df['order_date']).dt.days # today MINUS date = positive
NeedSyntax
Parse strings to datetimepd.to_datetime(col, errors='coerce')
Get today's timestamppd.Timestamp.now()
Extract year/month/daydf['date'].dt.year / .dt.month / .dt.day
Day-of-weekdf['date'].dt.dayofweek / .dt.day_name()
Difference between two dates(date1 - date2).dt.days
Filter last N daysdf['date'] >= pd.Timestamp.now() - pd.Timedelta(days=30)
💡

Subtracting two datetime columns gives a Timedelta, not a number — that's why .dt.days is needed to pull out the integer day count.

Interview Trap

Getting the subtraction order backwards. (order_date - today) gives a negative number for past orders. For "days since X" (a positive count going forward), it's always today - past_date.

GroupBy, Agg, Transform & Rank

MethodOutput shapeWhen to use
.groupby().agg()One row per groupSummary table (e.g. avg price per category)
.groupby().transform()Same shape as original dfAdd group stat back onto every row (e.g. "price vs category avg")
.groupby().rank()Same shape, ranks within groupRank rows within their own group
.groupby().nunique()One row per groupCount distinct values (e.g. distinct users), not just rows
# transform -- broadcasts the group mean back onto every row df['category_avg'] = df.groupby('category')['price'].transform('mean') df['price_vs_avg'] = df['price'] / df['category_avg']
💡

"Top X within a group" is a two-step problem. Grouping by too many keys at once (e.g. region + salesperson) collapses the exact comparison you need. Aggregate first (one row per entity), then rank the aggregated table within the outer group.

# WRONG: groups by region+salesperson together -- never compares people to each other df.groupby(['region','salesperson'])['revenue'].rank(...) # RIGHT: aggregate first, then rank across salespeople within region agg = df.groupby(['region','salesperson'])['revenue'].sum().reset_index() agg['rank'] = agg.groupby('region')['revenue'].rank(method='first', ascending=False) top = agg[agg['rank'] == 1]
Quick Check
Q. Why use nunique() instead of count() for "number of users who viewed"?
A. count() counts rows/events — if one user views a product twice, they'd be counted twice. nunique() counts distinct users, matching the actual question.
Q. Why can't you chain .reset_index() after sort_values(..., inplace=True)?
A. inplace=True modifies the dataframe and returns None — there's nothing to chain onto. Either drop inplace and reassign, or split into two lines.

Merge, Duplicates & Nulls

df = views.merge(cart_adds, on=['user_id','product_id'], how='left', suffixes=('_view','_cart'))

Three ways to dedupe, keeping the latest row per key

# Option A: sort + drop_duplicates (cleanest) df = df.sort_values('updated_at').drop_duplicates(subset='listing_id', keep='last') # Option B: idxmax df = df.loc[df.groupby('listing_id')['updated_at'].idxmax()] # Option C: rank (more verbose, same result) df['rank'] = df.groupby('listing_id')['updated_at'].rank(method='first', ascending=False) df = df[df['rank'] == 1]
💡

dropna(subset=['price']) only drops rows where that specific column is null — plain dropna() with no subset drops a row if any column has a null, which is usually not what you want.

Interview Trap

Merging two raw event tables (views × cart_adds) and then using .count() on timestamps to get "users who viewed/added" — a merge fans out rows on duplicate keys, inflating your counts. Aggregate each side to distinct users before merging, or use nunique() directly.

Pivot Tables

pd.pivot_table(df, values='revenue', index='region', columns='month', aggfunc='sum')
💡

Default aggfunc is 'mean', not 'sum'. If the question asks for "total," you must pass aggfunc='sum' explicitly. Also — pandas uses 'mean', never 'avg' (that's SQL syntax bleeding in).

Interview Trap

Passing values/index/columns positionally instead of as keywords. It works, but is easy to mix up under time pressure and unclear to a reader — always write values=, index=, columns= explicitly.

Conditional Logic & Filtering

SituationCorrect syntax
Combining boolean Series conditions(cond1) & (cond2) — never and/or
If/else creating a new columnnp.where(condition, value_if_true, value_if_false)
Extracting dict values from a columndf['col'].apply(lambda x: x.get('key')) or .str.get('key')
# WRONG -- 'and' only works on single booleans, not Series df[df['a'] >= 1 and df['b'] <= 2] # RIGHT df[(df['a'] >= 1) & (df['b'] <= 2)]
Interview Trap

np.where(df[boolean_mask], ...) — wrapping the condition in df[...] filters rows instead of passing a condition to evaluate. np.where takes the boolean Series directly, no df[] wrapper.

Cheat Sheet

Common Traps Quick Reference

Syntax Traps
  • and/or on Series → use & / | with parens
  • String methods need .str. accessor — never call directly on a column
  • 'avg' is not valid — pandas uses 'mean'
  • errors='coerce', not coerce_errors=True
  • inplace=True returns None — can't chain after it
  • reset_index(drop=True) — forgetting drop=True adds a stray column
Logic Traps
  • .str.contains() takes one pattern — join a list with '|' for "any of"
  • count() counts rows/events; nunique() counts distinct entities — pick deliberately
  • "Top X per group" often needs aggregate-then-rank, not rank on raw rows
  • today − past_date is positive; past_date − today is negative — order matters
  • Merging raw event tables before counting can silently inflate counts (fan-out)
  • np.where(condition, a, b) — condition is a boolean Series, not df[condition]

Practice Questions & Answers

S1Easy
Extract the top-level category from "Electronics/Phones/Smartphones".
+
Answer
df['main_category'] = df['category_path'].str.split('/').str[0]

Common trap: calling .split() directly on a column instead of via .str.

S2Medium
Extract 'brand' from a dict column like {'brand':'Nike','color':'red'}.
+
Answer
products['brand'] = products['attributes'].apply(lambda x: x.get('brand')) # or: products['attributes'].str.get('brand')

Using .get() avoids a KeyError if a row's dict is missing that key. Indexing the Series directly with a key (series['brand']) is a common trap.

S3Easy
Filter orders where status=='shipped' and product_name contains "phone" (case-insensitive).
+
Answer
df[(df['status'] == 'shipped') & (df['product_name'].str.contains('phone', case=False, na=False))]

case=False makes an extra .str.lower() unnecessary; na=False prevents nulls from breaking the boolean mask.

S4Medium
Strip a phone column like "(123) 456-7890" down to just digits.
+
Answer
df['phone_clean'] = df['phone'].str.replace(r'\D', '', regex=True)

\D matches any non-digit character; regex=True tells pandas to treat the pattern as regex, not literal text.

S5Medium
Extract the domain from an email column, e.g. "jane@ebay.com" → "ebay.com".
+
Answer
df['domain'] = df['email'].str.split('@').str[1] # or, more robust to malformed emails: df['domain'] = df['email'].str.extract(r'@(.+)$')

.str.extract() with a regex capture group returns NaN instead of erroring on rows with no "@" — safer on messy real-world data than a plain split.

S6Easy
Count the number of words in a product title column.
+
Answer
df['word_count'] = df['title'].str.split().str.len()

.str.split() with no argument splits on any whitespace and collapses multiple spaces — safer than .str.split(' '), which would count consecutive spaces as empty tokens.

S7Easy
Standardize a 'city' column: trim whitespace and convert to title case.
+
Answer
df['city'] = df['city'].str.strip().str.title()

Each .str. call returns a new Series, so chaining .str.strip().str.title() requires the accessor before every method — a single .str.strip().title() would fail.

S8Hard
Flag SKUs that don't match the expected pattern: 3 letters followed by 4 digits (e.g. "ABC1234").
+
Answer
df['invalid_sku'] = ~df['sku'].str.match(r'^[A-Za-z]{3}\d{4}$')

.str.match() anchors at the start of the string by default (like re.match); adding $ also anchors the end so partial matches on longer strings aren't accepted. The ~ inverts the mask to flag invalid rows.

D1Hard
Convert order_date string to datetime, add days_since_order relative to today.
+
Answer
today = pd.Timestamp.now() df['order_date'] = pd.to_datetime(df['order_date'], errors='coerce') df['days_since_order'] = (today - df['order_date']).dt.days

Common traps: wrong parameter name (coerce_errors vs errors='coerce'), and reversing the subtraction order, which flips the sign to negative.

D2Easy
Extract the month name and day-of-week name from a datetime column.
+
Answer
df['month_name'] = df['order_date'].dt.month_name() df['day_of_week'] = df['order_date'].dt.day_name()

Both return human-readable strings ("January", "Monday") — use .dt.month / .dt.dayofweek instead if you need the numeric versions for sorting or modeling.

D3Medium
Filter to orders placed on a weekend.
+
Answer
df[df['order_date'].dt.dayofweek >= 5]

.dt.dayofweek returns 0=Monday through 6=Sunday, so 5 and 6 are Saturday/Sunday.

D4Easy
Compute age in years from a date_of_birth column.
+
Answer
df['age'] = ((pd.Timestamp.now() - df['dob']).dt.days / 365.25).astype(int)

Dividing by 365.25 (not 365) accounts for leap years over long spans. .astype(int) truncates to whole years rather than rounding.

D5Medium
Bucket each order into "this_week", "this_month", or "older" based on order_date.
+
Answer
now = pd.Timestamp.now() conditions = [ df['order_date'] >= now - pd.Timedelta(days=7), df['order_date'] >= now - pd.Timedelta(days=30) ] choices = ['this_week', 'this_month'] df['recency_bucket'] = np.select(conditions, choices, default='older')

np.select is the right tool once you have more than two outcomes — np.where only handles binary if/else.

D6Medium
Find each user's first and most recent order date.
+
Answer
df.groupby('user_id')['order_date'].agg(['min', 'max']).reset_index()

min/max work directly on datetime columns inside .agg() — no special date-handling needed since datetimes are naturally ordered.

D7Hard
Compute a 7-day rolling average of daily revenue.
+
Answer
daily = df.groupby('order_date')['revenue'].sum().reset_index().sort_values('order_date') daily['rolling_7d_avg'] = daily['revenue'].rolling(window=7).mean()

Data must be sorted by date and aggregated to one row per day first — .rolling() operates on row order, not on the date values themselves, so unsorted or duplicate-date rows silently give wrong results.

D8Medium
Convert a Unix timestamp column (epoch seconds) into a readable datetime.
+
Answer
df['event_time'] = pd.to_datetime(df['epoch_seconds'], unit='s')

unit='s' tells pandas the numbers are seconds since 1970-01-01 — forgetting it makes pandas assume nanoseconds (its default), producing dates in the year 1970.

G1Medium
Top 3 users by total spend in the last 30 days.
+
Answer
df_2 = df[(df['order_date'] >= '2026-06-21') & (df['order_date'] <= '2026-07-22')] user_spends = df_2.groupby('user_id')['spend'].sum().reset_index() user_spends = user_spends.sort_values('spend', ascending=False).reset_index(drop=True) user_spends.head(3)

Common traps: and instead of & for combining conditions, and chaining a method after sort_values(..., inplace=True), which returns None.

G2Easy
Add price_vs_category_avg = price / avg price of its category, in one line.
+
Answer
df['price_vs_category_avg'] = df['price'] / df.groupby('category')['price'].transform('mean')

.transform() broadcasts the group stat back onto every row in the original shape — .agg() would collapse to one row per group instead.

G3Hard
Top salesperson (by total revenue) within each region.
+
Answer
agg = df.groupby(['region', 'salesperson'])['revenue'].sum().reset_index() agg['rank'] = agg.groupby('region')['revenue'].rank(method='first', ascending=False) top_sellers = agg[agg['rank'] == 1]

Ranking directly on groupby(['region','salesperson']) compares each person only to their own rows. Aggregate to one row per salesperson first, then rank within region.

G4Hard
Cancellation rate per user = cancelled orders / total orders.
+
Answer
cancel_counts = df[df['status'] == 'cancelled'].groupby('user_id')['order_id'].count() total_counts = df.groupby('user_id')['order_id'].count() cancellation_rate = (cancel_counts / total_counts).fillna(0).reset_index(name='cancellation_rate')

A row-level np.where approach only produces a meaningful value on 'cancelled' rows — aggregating each piece separately, then dividing, gives one clean rate per user.

G5Easy
Find users who have placed more than 5 orders.
+
Answer
order_counts = df.groupby('user_id')['order_id'].count() frequent_users = order_counts[order_counts > 5].reset_index(name='order_count')

Filtering happens on the aggregated Series before resetting the index — filtering the raw dataframe first would not give per-user totals.

G6Medium
What % of total revenue does each category contribute?
+
Answer
cat_revenue = df.groupby('category')['revenue'].sum().reset_index() cat_revenue['pct_of_total'] = cat_revenue['revenue'] / cat_revenue['revenue'].sum() * 100

The denominator (.sum() of the already-aggregated column) is the grand total across categories — dividing by the ungrouped raw column would give the wrong denominator.

G7Hard
Flag rows where a value is a statistical outlier relative to its group (z-score > 3).
+
Answer
group_mean = df.groupby('category')['price'].transform('mean') group_std = df.groupby('category')['price'].transform('std') df['z_score'] = (df['price'] - group_mean) / group_std df['is_outlier'] = df['z_score'].abs() > 3

Two separate .transform() calls (mean and std) are needed since .agg(['mean','std']) would collapse to one row per group instead of broadcasting to every row.

G8Easy
Find the number of distinct products each user has purchased.
+
Answer
df.groupby('user_id')['product_id'].nunique().reset_index(name='distinct_products')

nunique() counts distinct values, not rows — using count() here would double-count a user who bought the same product twice.

M1Medium
Per-product click-through rate = users who added / users who viewed.
+
Answer
views_count = views.groupby('product_id')['user_id'].nunique().reset_index(name='users_who_viewed') adds_count = cart_adds.groupby('product_id')['user_id'].nunique().reset_index(name='users_who_added') df_2 = views_count.merge(adds_count, on='product_id', how='left') df_2['users_who_added'] = df_2['users_who_added'].fillna(0) df_2['click_through_rate'] = df_2['users_who_added'] / df_2['users_who_viewed']

Aggregate each side to distinct users before merging — merging raw event tables first and counting rows afterward causes a fan-out that inflates the counts.

M2Medium
Dedupe listings: one row per listing_id (latest by updated_at), rows with null price dropped.
+
Answer
df = df.dropna(subset=['price']) df = df.sort_values('updated_at').drop_duplicates(subset='listing_id', keep='last') # or: df.loc[df.groupby('listing_id')['updated_at'].idxmax()]

dropna(subset=['price']) only drops rows where that specific column is null — plain dropna() would drop a row if any column had a null.

M3Medium
Find users present in the "signups" table but not in the "orders" table (i.e., never ordered).
+
Answer
merged = signups.merge(orders[['user_id']].drop_duplicates(), on='user_id', how='left', indicator=True) never_ordered = merged[merged['_merge'] == 'left_only']

indicator=True adds a _merge column showing whether each row matched 'both', 'left_only', or 'right_only' — the cleanest way to do an anti-join in pandas.

M4Easy
Combine an orders table and a customers table, keeping only rows that match in both.
+
Answer
df = orders.merge(customers, on='customer_id', how='inner')

how='inner' (the default) keeps only matching keys — use 'left' to keep all orders even if a customer record is missing.

M5Easy
Find duplicate rows based only on ['user_id', 'product_id'], ignoring other columns.
+
Answer
dupes = df[df.duplicated(subset=['user_id', 'product_id'], keep=False)]

keep=False flags all occurrences of a duplicate (not just the 2nd onward) — useful when you want to inspect every duplicated row, not just remove extras.

M6Medium
Fill missing values in a 'category' column using values from a backup table, matched by product_id.
+
Answer
df = df.merge(backup[['product_id','category']], on='product_id', how='left', suffixes=('', '_backup')) df['category'] = df['category'].fillna(df['category_backup']) df = df.drop(columns=['category_backup'])

suffixes prevents a column name collision on merge; fillna then only replaces the nulls, leaving existing values untouched.

M7Easy
Combine 12 monthly CSV files into a single dataframe.
+
Answer
import glob files = glob.glob('sales_2026_*.csv') df = pd.concat([pd.read_csv(f) for f in files], ignore_index=True)

ignore_index=True re-indexes the combined dataframe from 0 — without it, each file's original index is preserved, causing duplicate index values across files.

M8Medium
Merge two tables where the join key is named differently in each (e.g. 'user_id' vs 'customer_id').
+
Answer
df = orders.merge(customers, left_on='user_id', right_on='customer_id', how='left')

left_on/right_on are needed instead of on whenever the key names differ — the result keeps both original key columns unless you explicitly drop one.

P1Easy
Pivot table: region as rows, month as columns, total revenue as values.
+
Answer
pd.pivot_table(df, values='revenue', index='region', columns='month', aggfunc='sum')

Default aggfunc is 'mean', not 'sum' — for totals it must be passed explicitly. Also note pandas uses 'mean', never 'avg'.

P2Easy
Build a table showing average and max bid_price per category.
+
Answer
pd.pivot_table(df, values='bid_price', index='category', aggfunc=['mean', 'max'])

Passing a list to aggfunc computes multiple statistics at once, producing a multi-level column header (e.g. ('mean','bid_price'), ('max','bid_price')).

P3Medium
Build a pivot table showing both total revenue and order count per region/month.
+
Answer
pd.pivot_table(df, values='revenue', index='region', columns='month', aggfunc=['sum', 'count'])

Same pattern as combining aggregations in .agg() — pass a list of functions instead of one.

P4Medium
Add grand total row/column to a region x month revenue pivot table.
+
Answer
pd.pivot_table(df, values='revenue', index='region', columns='month', aggfunc='sum', margins=True, margins_name='Total')

margins=True adds an "All"/"Total" row and column with grand totals — margins_name lets you rename it from the default 'All'.

P5Hard
Convert a wide pivoted table (region rows, month columns) back into long format.
+
Answer
long_df = wide_df.reset_index().melt(id_vars='region', var_name='month', value_name='revenue')

.melt() is the inverse of pivoting — id_vars specifies which column(s) stay fixed, and every other column gets unpivoted into name/value pairs.

P6Medium
Build a pivot table with a two-level row index: category then subcategory.
+
Answer
pd.pivot_table(df, values='revenue', index=['category', 'subcategory'], columns='month', aggfunc='sum')

Passing a list to index creates a hierarchical (multi-level) row index — access a group with .loc[('Electronics','Phones')].

P7Easy
A pivot table has NaN cells for region/month combos with no data. Replace them with 0.
+
Answer
pd.pivot_table(df, values='revenue', index='region', columns='month', aggfunc='sum', fill_value=0)

fill_value is a pivot_table parameter that handles this at creation time — cleaner than a separate .fillna(0) call afterward.

P8Medium
Get a count of orders broken down by status (rows) and region (columns).
+
Answer
pd.crosstab(df['status'], df['region'])

pd.crosstab() is a shortcut for pivot tables whose only purpose is counting frequency — no values/aggfunc needed since counting is its default behavior.

L1Medium
Flag listings 'high_value' if title contains diamond/gold/luxury, else 'standard'.
+
Answer
lis = ['diamond', 'gold', 'luxury'] pattern = '|'.join(lis) df['category'] = np.where(df['title'].str.contains(pattern, case=False, na=False), 'high_value', 'standard')

.str.contains() takes a single pattern, not a list — join the list with '|' for regex OR. np.where takes the boolean Series directly, never wrapped in df[...].

L2Easy
Filter rows where price is between 10 and 50 AND category is 'Electronics'.
+
Answer
df[(df['price'] >= 10) & (df['price'] <= 50) & (df['category'] == 'Electronics')]

and/or only work on single booleans, not Series — combining conditions on columns always requires &/| with each condition parenthesized.

L3Medium
Classify orders into 'high', 'medium', 'low' spend tiers based on order value (3+ outcomes).
+
Answer
conditions = [df['order_value'] >= 500, df['order_value'] >= 100] choices = ['high', 'medium'] df['spend_tier'] = np.select(conditions, choices, default='low')

np.where only handles two outcomes (if/else) — once there are 3+ categories, np.select is the right tool. Conditions are checked in order; the first match wins.

L4Easy
Set all negative values in a 'quantity' column to 0.
+
Answer
df.loc[df['quantity'] < 0, 'quantity'] = 0

.loc[row_condition, column] is the standard pattern for conditionally overwriting values in place — safer than chained indexing like df[df['quantity']<0]['quantity']=0, which can trigger a SettingWithCopyWarning and silently fail.

L5Medium
Create a column combining two others with custom row-wise logic (e.g. a discount rule based on both price and category).
+
Answer
def discount_rule(row): if row['category'] == 'Electronics' and row['price'] > 500: return 0.10 return 0.0 df['discount'] = df.apply(discount_rule, axis=1)

axis=1 applies the function row-wise (giving access to multiple columns via row[...]) instead of the default axis=0, which applies column-wise. Note: row-wise apply is slow on large data — vectorized np.where/np.select is preferred when the logic allows it.

L6Medium
Cap ('clip') a 'price' column so no value exceeds 1000 or falls below 0.
+
Answer
df['price'] = df['price'].clip(lower=0, upper=1000)

.clip() is purpose-built for this — cleaner than two separate .loc[] conditional overwrites for the same job.

L7Easy
Flag orders where category is one of ['Books','Toys','Games'].
+
Answer
df['is_target_category'] = df['category'].isin(['Books', 'Toys', 'Games'])

.isin() is the vectorized way to check membership in a list — avoids writing out a chain of (df['category']=='Books') | (df['category']=='Toys') | ....

L8Hard
Flag only each user's first purchase (chronologically) as True, all later purchases as False.
+
Answer
df = df.sort_values(['user_id', 'order_date']) df['is_first_purchase'] = ~df.duplicated(subset='user_id', keep='first')

Sort by date first so "first" reflects chronological order, not row order in the raw data. duplicated(keep='first') marks every row after the first occurrence as True, so negating with ~ flags only the first purchase itself.
