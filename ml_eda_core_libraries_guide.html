Step 1

Data Loading & First Look

Before any analysis, you need to load the data and get an honest first impression: how big is it, what types are the columns, is anything obviously broken?

import pandas as pd df = pd.read_csv("data.csv") # loads a CSV into a DataFrame (a table-like object) df.head() # first 5 rows -- quick visual sanity check df.shape # (rows, columns) -- e.g. (10000, 15) df.info() # column names, dtypes, non-null counts, memory usage -- all in one call df.describe() # count/mean/std/min/max/quartiles for NUMERIC columns only df.dtypes # just the data type of each column df.sample(5) # 5 RANDOM rows -- better than head() for spotting hidden issues

What does each piece mean?

  • DataFrame — pandas' main 2D table structure (rows + labeled columns), like a spreadsheet you can code against
  • .info() — tells you the dtype of every column and how many non-null values exist, which is the fastest way to spot missing data at a glance
  • .describe() — by default only summarizes numeric columns; pass include='all' to also see categorical column stats (unique count, most frequent value)
💡

df.head() only shows the top of the file — if your data was sorted or exported in a particular order, the first 5 rows can look deceptively clean. Always pair it with df.sample() and df.info().

Quick Check — Data Loading
Q. If a numeric-looking column shows up as "object" dtype in df.info(), what does that usually mean?
A. It likely contains a mix of types or stray characters (e.g. commas, currency symbols, or a stray text value) preventing pandas from parsing it as a number — worth checking with df['col'].unique().
Q. Why might df.describe() alone miss important issues in a dataset?
A. It only summarizes numeric columns by default and shows aggregate statistics — it won't reveal missing values, duplicate rows, or categorical imbalances on its own.
Interview Trap

Relying only on df.head() to judge whether a dataset is "clean." Real datasets often have issues concentrated at the end, in random rows, or in specific columns that a 5-row preview won't reveal.

Step 2

Handling Missing Data

df.isnull().sum() # count of missing values PER COLUMN df.isna().mean() * 100 # PERCENTAGE missing per column -- often more useful than raw counts df.dropna() # drops any ROW with at least one missing value df.dropna(axis=1) # drops any COLUMN with at least one missing value df.dropna(thresh=5) # keeps rows with at least 5 non-null values df['age'].fillna(df['age'].mean(), inplace=True) # impute with the mean df['category'].fillna(df['category'].mode()[0], inplace=True) # impute categorical with the most frequent value df['value'].fillna(method='ffill', inplace=True) # forward-fill: carry the last valid value forward

Smarter imputation with scikit-learn

from sklearn.impute import SimpleImputer, KNNImputer imputer = SimpleImputer(strategy='median') # median is more robust to outliers than mean df[['age']] = imputer.fit_transform(df[['age']]) knn_imputer = KNNImputer(n_neighbors=5) # fills missing values based on similar rows (neighbors) df_filled = knn_imputer.fit_transform(df)

Why the MISSING PATTERN matters, not just the count

TypeMeaning
MCARMissing Completely At Random — missingness has nothing to do with any value; safest to drop or simple-impute
MARMissing At Random — missingness depends on OTHER observed columns (e.g. income missing more often for younger respondents); model-based imputation works well
MNARMissing Not At Random — missingness depends on the missing value ITSELF (e.g. high earners refusing to disclose income); hardest to fix, may need domain flags instead of imputation
Quick Check — Missing Data
Q. Why is median often preferred over mean for imputing a skewed numeric column?
A. The mean is pulled toward outliers/skew, so it can misrepresent the "typical" value; the median is robust to extreme values and better reflects the center of a skewed distribution.
Q. What's the risk of just dropping all rows with any missing value?
A. You can lose a large chunk of your dataset (especially if missingness is spread across many columns), and if missingness isn't random (MAR/MNAR), dropping rows can bias your remaining sample.
Interview Trap

Imputing BEFORE splitting into train/test sets — this leaks information from the test set into training (the imputed mean/median is computed using data the model shouldn't have seen yet). Always fit the imputer on the training set only, then apply it to the test set.

Step 3

Data Cleaning — Duplicates, Types, Outliers

df.duplicated().sum() # count of fully duplicate rows df.drop_duplicates(inplace=True) # removes them, keeps the first occurrence by default df['date'] = pd.to_datetime(df['date']) # parses strings into real datetime objects df['price'] = df['price'].str.replace(',', '').astype(float) # strip commas, then convert to float df['name'] = df['name'].str.strip().str.lower() # remove whitespace, standardize casing

Outlier detection — IQR method

Q1 = df['income'].quantile(0.25) Q3 = df['income'].quantile(0.75) IQR = Q3 - Q1 lower = Q1 - 1.5 * IQR upper = Q3 + 1.5 * IQR outliers = df[(df['income'] < lower) | (df['income'] > upper)] print(f"Found {len(outliers)} outliers")

Outlier detection — Z-score method

from scipy import stats import numpy as np z_scores = np.abs(stats.zscore(df['income'])) outliers = df[z_scores > 3] # conventionally, |z| > 3 is flagged as an outlier
💡

IQR is more robust for skewed data (it doesn't assume a normal distribution); Z-score assumes roughly normal data and is sensitive to the very outliers you're trying to detect (since they inflate the mean/std used to compute it).

Quick Check — Cleaning
Q. Why use 1.5 × IQR specifically as the outlier boundary?
A. It's a widely-used convention (originating from Tukey's boxplot method) that, for roughly normal data, flags a reasonable minority of points as outliers without being overly aggressive — it's a convention, not a hard mathematical law.
Q. Is an "outlier" always something you should remove?
A. No — sometimes outliers are the most important data points (e.g. fraud detection, rare disease cases). Always investigate whether it's a data error or a genuine, meaningful extreme value before deciding.
Interview Trap

df.drop_duplicates() checks ALL columns by default — two rows with the same customer ID but a different timestamp won't be flagged as duplicates unless you specify subset=['customer_id'].

Step 4

Pandas Core Operations

.loc vs .iloc

df.loc[0, 'name'] # access by LABEL -- row label 0, column named 'name' df.iloc[0, 1] # access by INTEGER POSITION -- 1st row, 2nd column, regardless of labels df.loc[df['age'] > 30] # boolean filtering -- keep only rows where age > 30

groupby — the single most-tested pandas skill

df.groupby('department')['salary'].mean() # average salary PER department df.groupby('department').agg( # multiple aggregations at once avg_salary=('salary', 'mean'), headcount=('salary', 'count') ) df.groupby(['department', 'level'])['salary'].mean() # group by MULTIPLE columns

merge / join

pd.merge(orders, customers, on='customer_id', how='left') # how='left' -> keep all rows from orders, matching customer info where it exists # how='inner' -> keep only rows with a match in BOTH tables # how='outer' -> keep everything from both, NaN where there's no match

pivot_table

df.pivot_table(values='sales', index='region', columns='quarter', aggfunc='sum') # reshapes long data into a region x quarter grid of summed sales
💡

Prefer vectorized pandas/numpy operations (df['col'] * 2) over .apply() with a Python function, and avoid row-by-row for loops entirely — vectorized operations run in fast compiled C code, while .apply()/loops run slow, interpreted Python per row.

Quick Check — Pandas Core Ops
Q. What's the difference between df.loc[2] and df.iloc[2] if the DataFrame's index has been shuffled/reset unusually?
A. df.loc[2] looks for the row LABELED 2 (which may not be the 3rd physical row anymore), while df.iloc[2] always gets the 3rd physical row (0-indexed), regardless of labels.
Q. Why might a left join produce more rows than the original left table?
A. If the right table has multiple matching rows for a single key (a one-to-many relationship), the left join duplicates the left row once per match — always check for unexpected row count changes after a merge.
Interview Trap

Chained indexing like df[df['age'] > 30]['salary'] = 0 can silently fail to modify the original DataFrame (triggers a SettingWithCopyWarning) because it's ambiguous whether you're working on a view or a copy. Use df.loc[df['age'] > 30, 'salary'] = 0 instead.

Step 5

Univariate Analysis — Looking at One Column at a Time

import seaborn as sns import matplotlib.pyplot as plt sns.histplot(df['income'], kde=True) # histogram + smoothed density curve, for numeric columns plt.show() sns.countplot(x='department', data=df) # bar chart of category FREQUENCIES plt.show() df['income'].skew() # > 0: right-skewed (long tail to the right); < 0: left-skewed df['income'].kurtosis() # > 0: heavier tails / more outliers than a normal distribution
💡

Skewness tells you WHICH direction a distribution leans — a right-skewed income column (common in real data) means most people earn near the low end with a few very high earners pulling the mean upward. This is exactly why median often beats mean for such columns.

Quick Check — Univariate Analysis
Q. If a histogram of income is heavily right-skewed, would you use the mean or median to describe the "typical" income?
A. Median — it's not distorted by the small number of very high earners the way the mean is.
Q. What plot would you use to see the frequency of each category in a categorical column?
A. A count plot / bar chart (sns.countplot), not a histogram — histograms are for numeric/continuous data.
Interview Trap

Applying a histogram to a categorical column encoded as numbers (e.g. a 1-5 rating scale) treats it as continuous data, potentially hiding that only 5 discrete values actually exist. Check .nunique() first before choosing a plot type.

Bivariate & Multivariate Analysis

Numerical vs Numerical

sns.scatterplot(x='age', y='income', data=df) plt.show() df[['age', 'income']].corr() # Pearson correlation coefficient between the two columns

Categorical vs Numerical

sns.boxplot(x='department', y='salary', data=df) # compare salary DISTRIBUTION across departments plt.show() df.groupby('department')['salary'].agg(['mean', 'median', 'std'])

Categorical vs Categorical

pd.crosstab(df['department'], df['gender']) # frequency table -- counts per combination pd.crosstab(df['department'], df['gender'], normalize='index') # row percentages instead of raw counts

Multivariate — correlation heatmap

corr_matrix = df.corr(numeric_only=True) sns.heatmap(corr_matrix, annot=True, cmap='coolwarm') plt.show()
Quick Check — Bivariate Analysis
Q. Which plot would you use to compare the salary distribution across 5 different departments?
A. A boxplot (or violin plot) — it shows median, spread, and outliers for each category side by side, which a simple bar chart of means would hide.
Q. What does pd.crosstab with normalize='index' actually compute?
A. It converts each row's counts into percentages that sum to 1 across that row — useful for comparing proportions (e.g. "what % of each department is each gender") rather than raw counts.
Interview Trap

Correlation only captures LINEAR relationships. Two variables can have a strong non-linear (e.g. U-shaped) relationship and still show a correlation near 0 — always pair a correlation matrix with actual scatter plots before concluding "no relationship."

Step 7

Feature Engineering

Encoding categorical variables

pd.get_dummies(df, columns=['department'], drop_first=True) # one-hot encoding; drop_first avoids redundancy from sklearn.preprocessing import LabelEncoder le = LabelEncoder() df['dept_encoded'] = le.fit_transform(df['department']) # assigns each category an integer -- careful, implies false ordering

Scaling numeric features

from sklearn.preprocessing import StandardScaler, MinMaxScaler, RobustScaler scaler = StandardScaler() # mean=0, std=1 -- good default, sensitive to outliers df[['income']] = scaler.fit_transform(df[['income']]) scaler = MinMaxScaler() # squashes values into [0, 1] -- good for bounded features scaler = RobustScaler() # uses median/IQR instead of mean/std -- robust to outliers

Handling skewed numeric data

import numpy as np df['income_log'] = np.log1p(df['income']) # log1p = log(1+x), handles zero values safely, reduces right-skew

Feature creation from dates

df['day_of_week'] = df['date'].dt.dayofweek # 0=Monday ... 6=Sunday df['is_weekend'] = df['day_of_week'].isin([5, 6]) df['month'] = df['date'].dt.month
💡

Label encoding turns categories into numbers (0, 1, 2...) which implies a false ORDER to models that assume numeric meaning (like linear regression). One-hot encoding avoids that by giving each category its own binary column — but explodes column count for high-cardinality features (many unique categories).

Quick Check — Feature Engineering
Q. Why use drop_first=True in one-hot encoding?
A. To avoid the "dummy variable trap" — with N categories, only N-1 dummy columns are needed since the last category is implied when all others are 0, avoiding redundant/collinear columns.
Q. When would you prefer RobustScaler over StandardScaler?
A. When your data has significant outliers — StandardScaler uses mean/std, which outliers distort heavily; RobustScaler uses median/IQR, which stays stable even with extreme values present.
Interview Trap

Fitting a scaler or encoder on the FULL dataset (train + test combined) before splitting leaks test-set statistics into training. Always fit_transform() on training data only, then transform() (not fit again) on the test set.

Step 8

Correlation & Multicollinearity

MethodUse case
PearsonLinear relationships between two continuous variables (most common default)
SpearmanMonotonic (consistently increasing/decreasing, not necessarily linear) relationships; works on ranked data, robust to outliers
KendallSimilar to Spearman, often preferred for small sample sizes
df[['age', 'income']].corr(method='pearson') df[['age', 'income']].corr(method='spearman')

Multicollinearity — VIF (Variance Inflation Factor)

from statsmodels.stats.outliers_influence import variance_inflation_factor X = df[['age', 'income', 'education_years']] vif_data = pd.DataFrame() vif_data['feature'] = X.columns vif_data['VIF'] = [variance_inflation_factor(X.values, i) for i in range(X.shape[1])] # VIF > 5 (some say 10) suggests that feature is highly explainable by the OTHER features -- multicollinearity risk
💡

Multicollinearity doesn't hurt every model equally — it destabilizes coefficient interpretation in linear/logistic regression, but tree-based models (random forest, gradient boosting) are largely unaffected by it.

Quick Check — Correlation
Q. Two variables have a Pearson correlation of 0.02 but a Spearman correlation of 0.85. What does this suggest?
A. There's likely a strong monotonic but non-linear relationship (e.g. exponential) — Pearson only detects LINEAR patterns and misses it, while Spearman (based on ranks) captures the consistent increasing trend.
Q. Why does high multicollinearity matter more for linear regression than for random forest?
A. Linear regression estimates individual coefficients that become unstable/unreliable when predictors are highly correlated with each other; random forest splits on one feature at a time and doesn't rely on stable individual coefficients in the same way.
Interview Trap

"Correlation implies causation" — a classic trap phrase interviewers listen for. A high correlation between two variables never proves one causes the other; there could be a confounding third variable, reverse causation, or pure coincidence.

Step 9

Statistical Testing in EDA

from scipy import stats # t-test: compare MEANS of a numeric variable between 2 groups group_a = df[df['group'] == 'A']['score'] group_b = df[df['group'] == 'B']['score'] t_stat, p_value = stats.ttest_ind(group_a, group_b) # chi-square test: check ASSOCIATION between two categorical variables contingency = pd.crosstab(df['gender'], df['purchased']) chi2, p_value, dof, expected = stats.chi2_contingency(contingency) # ANOVA: compare MEANS across 3+ groups at once f_stat, p_value = stats.f_oneway( df[df['dept']=='Sales']['salary'], df[df['dept']=='Eng']['salary'], df[df['dept']=='HR']['salary'] )
TestWhen to use
t-testCompare the mean of a numeric variable between exactly 2 groups
ANOVACompare means across 3 or more groups at once (avoids inflated error from running many t-tests)
Chi-squareTest whether two categorical variables are associated (independent or not)
💡

A p-value answers: "if there were truly NO effect/difference, how likely would we see data this extreme by chance?" A small p-value (typically < 0.05) suggests the observed difference is unlikely to be random noise — it does NOT mean "there's a 95% chance the effect is real," which is the most common misinterpretation.

Quick Check — Statistical Testing
Q. Why not just run multiple t-tests instead of one ANOVA when comparing 4 groups?
A. Running many pairwise t-tests inflates the chance of a false positive (multiple comparisons problem) — ANOVA tests all groups at once with one controlled error rate.
Q. A p-value of 0.03 came back for a correlation test. Does this prove the relationship is practically important?
A. No — statistical significance just means the effect is unlikely due to chance at your sample size; it says nothing about the effect's real-world magnitude or importance. Always check the effect size alongside the p-value.
Interview Trap

Treating p < 0.05 as a rigid, universal cutoff for "true" vs "false." It's a convention, not a law of nature — the right threshold depends on context, and a p-value just below 0.05 isn't meaningfully different from one just above it.

Quick Reference

Glossary of Every Term Used

DataFrame
Pandas' core 2D labeled table structure — rows and named columns, like a spreadsheet you can code against.
MCAR / MAR / MNAR
Categories describing WHY data is missing — completely random, related to other observed columns, or related to the missing value itself.
IQR (Interquartile Range)
The range between the 25th and 75th percentile of a numeric column — used as a robust outlier-detection threshold.
Z-score
How many standard deviations a value is from the mean — a common outlier-flagging metric for roughly normal data.
.loc / .iloc
Pandas indexers — .loc selects by label, .iloc selects by integer position.
groupby
Splits a DataFrame into groups based on a column's values, then applies an aggregation (mean, sum, count) to each group separately.
skewness
A measure of asymmetry in a distribution — positive means a longer tail on the right, negative means a longer tail on the left.
one-hot encoding
Converts a categorical column into multiple binary (0/1) columns, one per category, avoiding a false sense of numeric order.
multicollinearity
When two or more predictor variables are highly correlated with each other, destabilizing coefficient estimates in linear models.
VIF
Variance Inflation Factor — quantifies how much a feature's variance is inflated due to correlation with other features.
p-value
The probability of observing data this extreme (or more) if there were truly no real effect — small values suggest the effect isn't just random noise.
data leakage
When information from outside the training data (often the test set) improperly influences model training, inflating apparent performance.
Practice

50 Interview Questions

Conceptual Questions (Explain in Words)

C1Easy
Walk me through your general EDA process for a new dataset.
+
Answer
  • Load data, check shape/dtypes/first rows (df.info(), df.head())
  • Check for missing values and duplicates
  • Univariate analysis on each column (distributions, value counts)
  • Bivariate/multivariate analysis (correlations, groupby comparisons)
  • Feature engineering and cleaning based on findings
C2Easy
What's the difference between df.head() and df.sample()?
+
Answer

df.head() always shows the first N rows (in file order); df.sample() shows N random rows, which is better for catching issues that aren't concentrated at the start of the file.

C3Easy
What's the difference between .loc and .iloc?
+
Answer

.loc selects rows/columns by label (name), while .iloc selects by integer position, regardless of what the labels are.

C4Easy
Why would you drop duplicate rows in a dataset?
+
Answer

Duplicates can bias summary statistics and model training by over-weighting certain records, and often indicate a data collection or merge error worth investigating.

C5Easy
What's the difference between mean, median imputation, and when would you use each?
+
Answer

Mean imputation fills missing values with the column average; median uses the middle value. Median is preferred for skewed data or data with outliers, since the mean gets pulled by extreme values while the median stays stable.

C6Medium
Explain MCAR, MAR, and MNAR and why the distinction matters for handling missing data.
+
Answer

MCAR means missingness is unrelated to any value; MAR means it depends on other observed columns; MNAR means it depends on the missing value itself. The distinction matters because simple imputation is safe for MCAR/MAR but can introduce serious bias under MNAR, where the fact that data is missing is itself informative.

C7Medium
Why is data leakage a concern during EDA and preprocessing, not just modeling?
+
Answer

If you compute imputation values, scaling parameters, or encodings using the full dataset (including what will become the test set), those statistics carry information about the test set into training — inflating validation performance in a way that won't hold up on truly new data.

C8Medium
When would you use a boxplot over a histogram?
+
Answer

A boxplot is better for comparing the distribution (median, spread, outliers) of a numeric variable ACROSS multiple categories side by side; a histogram is better for seeing the detailed shape of a single distribution.

C9Medium
Why might you log-transform a skewed numeric feature before modeling?
+
Answer

Log transforms compress large values and stretch small ones, reducing right-skew and making the distribution closer to normal — which helps models (especially linear ones) that assume roughly normal, homoscedastic residuals.

C10Medium
What's the difference between Pearson and Spearman correlation?
+
Answer

Pearson measures linear relationships using the actual values; Spearman measures monotonic relationships using ranks, so it can catch consistent non-linear trends (and is more robust to outliers) that Pearson would miss or underestimate.

C11Medium
Why is one-hot encoding sometimes problematic for high-cardinality categorical features?
+
Answer

A feature with hundreds/thousands of unique categories (e.g. zip code) would explode into that many new columns, most of them sparse (mostly zeros) — increasing dimensionality and memory use dramatically. Alternatives include target encoding or grouping rare categories together.

C12Medium
What does a p-value actually tell you, and what's the most common misinterpretation?
+
Answer

It's the probability of observing data this extreme if there were truly no effect. The common misinterpretation is treating it as "the probability the null hypothesis is true" or "the probability the effect is real" — that's not what it measures.

C13Hard
Why can two variables have near-zero Pearson correlation but still be strongly related?
+
Answer

Pearson only captures LINEAR relationships. A strong non-linear relationship (e.g. U-shaped, or exponential) can produce a correlation coefficient near zero even though the variables are clearly and strongly related — always visualize with a scatter plot before trusting the number alone.

C14Hard
Why does multicollinearity matter for linear regression but much less for random forest?
+
Answer

Linear regression estimates a distinct coefficient per feature, and highly correlated features make those individual estimates unstable and hard to interpret. Tree-based models split on one feature at a time based on which reduces impurity most, and don't rely on stable coefficient estimates, so correlated features are much less damaging (though they can still dilute feature-importance scores across correlated features).

C15Hard
Why might you choose ANOVA over running multiple pairwise t-tests?
+
Answer

Running many pairwise t-tests inflates the overall chance of a false positive (the multiple comparisons problem) — each individual test has its own error rate that compounds. ANOVA tests all groups simultaneously under one controlled error rate.

C16Hard
How would you detect and reason about MNAR missingness in practice, since you can't directly observe the missing values?
+
Answer

You can't prove MNAR definitively from the data alone, but you can look for suspicious patterns — e.g. missingness correlating with other known-sensitive attributes, domain knowledge suggesting people with extreme values are likely to skip a question (like high earners not disclosing income). Adding a "was_missing" indicator flag as a feature can help the model account for this pattern.

C17Hard
Why is fitting a StandardScaler on the entire dataset before train/test split considered leakage?
+
Answer

The scaler's mean/std would be computed using test-set values, meaning the "unseen" test data has already influenced how the training data gets transformed — giving an overly optimistic sense of how the model performs on truly unseen data.

C18Hard
A dataset has 40% missing values in one column. What factors would guide whether you drop the column, impute it, or engineer a "missingness" flag?
+
Answer
  • How important the feature is expected to be for the target (domain knowledge)
  • Whether the missingness itself carries signal (MNAR) — if so, a flag may be more valuable than imputation
  • Whether other correlated features can reasonably substitute for it
C19Hard
Why is VIF calculated per feature rather than as one overall dataset-level number?
+
Answer

VIF measures how well ONE feature can be predicted from all the OTHER features — it's inherently a per-feature diagnostic. A single overall number couldn't tell you which specific feature is redundant and should be reconsidered or dropped.

C20Hard
How would you explain "correlation isn't causation" to a non-technical stakeholder using a concrete example?
+
Answer

"Ice cream sales and drowning incidents both rise in summer — they're correlated, but ice cream doesn't cause drowning. A hidden third factor (hot weather) drives both. Any correlation we find in this data could similarly be explained by something else entirely, so we shouldn't act on it as if we've proven cause and effect."

Why is this slower than a vectorized alternative, and how would you rewrite it?
