General Model Grid
Linear / Logistic RegressionHighLowRigid straight-line/plane boundary — underfits nonlinear patterns but is stable across resamples.
Ridge RegressionHigh (↑ vs OLS)Low (↓ vs OLS)L2 penalty shrinks coefficients toward zero — trades a bit of bias for a lot less variance.
Lasso RegressionHighLowL1 penalty shrinks and eliminates features — even more bias than Ridge, still low variance.
Naive BayesHighLowStrong conditional-independence assumption regardless of true feature correlation.
KNN (small k, e.g. k=1)LowVery HighPrediction depends on one nearest point — extremely sensitive to noise in training data.
KNN (large k)HighLowAveraging over many neighbors smooths the boundary until it oversimplifies.
SVM (linear kernel)HighLowSame rigidity as linear regression — a single hyperplane can't capture curvature.
SVM (RBF, poorly tuned C/gamma)LowHighA too-flexible kernel wraps tightly around every training point, including noise.
Decision Tree (deep / unpruned)LowVery HighSplits recursively until leaves are pure — effectively memorizes the training set.
Decision Tree (shallow / pruned)HighLowFew splits force an oversimplified, coarse decision surface.
Random ForestMed (~= base tree)Low (↓ vs single tree)Bagging averages many decorrelated high-variance trees, canceling out noise.
Gradient Boosting (early rounds)HighLowFew weak learners so far — hasn't captured enough signal yet.
Gradient Boosting (many rounds, no reg.)LowHigh (risk)Keeps fitting residuals indefinitely — can start memorizing noise without early stopping/shrinkage.
Neural Net (shallow / few epochs)HighLowLimited capacity or undertrained — hasn't learned the full function.
Neural Net (deep, no regularization)LowHighHuge parameter count lets it memorize training examples verbatim.
Tree-Family Pairs (A vs B)
Shallow Tree vs Deep TreeShallow → HigherDeep → HigherShallow stops early (underfits). Deep splits till pure (memorizes noise). Classic underfit/overfit pair.
Single Tree vs Random Forest~Similar (RF slightly ↑)Single Tree → HigherRF bags many trees and averages predictions — variance drops sharply, bias barely changes.
Random Forest vs Extra TreesRF slightly lowerExtra Trees lowerExtra Trees pick random split thresholds (not the best split) — more randomness, more decorrelation, tiny bias uptick.
Random Forest vs Gradient BoostingRF → HigherGBM → Higher (if untuned)RF = parallel bagging (variance reducer). GBM = sequential residual fitting (bias reducer) that can overfit unchecked.
AdaBoost (few iterations) vs AdaBoost (many iterations)Few → HigherMany → HigherMore rounds = more weak learners combined = closer fit to training data = variance creeps up.
Bagging vs Boosting (general)Bagging → HigherBoosting → Higher (if unregularized)Bagging tames variance of high-variance learners; boosting tames bias of high-bias learners.
Single Deep Tree vs GBM (shallow base learners)GBM's base learners → Higher individuallySingle Deep Tree → HigherGBM intentionally uses high-bias stumps/depth-3-4 trees and reduces bias iteratively; a lone deep tree is low-bias/high-variance.
XGBoost (default regularization) vs XGBoost (no reg., deep trees)Default reg → HigherNo reg → Highermax_depth / lambda / gamma intentionally trade a little bias for a lot less variance.

Linear Regression vs Decision Tree (deep)Linear Reg → HigherDecision Tree → HigherLinear assumes a straight-line relationship; tree splits recursively to fit any shape and is unstable to resampling.
Logistic Regression vs Decision TreeLogistic → HigherDecision Tree → HigherSame story for classification — linear boundary vs flexible axis-aligned splits.

KNN (k=1) vs Decision Tree (deep)~Both lowKNN(k=1) usually → HigherBoth memorize data, but KNN(k=1) has zero smoothing at all; the tree at least groups points via splits.

Summary Verdict
Mental test"If I retrained on a slightly different sample, would predictions change a lot?" → Yes = high variance. "Barely changes, but misses the real pattern?" → high bias.
Bagging vs Boosting ruleBagging (RF, Extra Trees) → base learner is high-variance (deep tree) → ensemble reduces VARIANCE. Boosting (AdaBoost, GBM, XGBoost) → base learner is high-bias (stump) → ensemble reduces BIAS.
Common Interview Pitfalls
"RF has zero bias"Wrong — RF inherits bias from its base trees; bagging only reduces variance, not bias.
"Boosting always has low bias AND low variance"Wrong — boosting reduces bias by design, but variance can rise again with too many rounds/no regularization (why XGBoost has lambda, gamma, max_depth).
"Complexity = variance, same word"Complexity causes variance, but they aren't interchangeable — a complex model can still be regularized to lower variance.
"Tree-based = always high variance"Wrong — a shallow tree (stump) is high bias. Depth is the deciding factor, not "is it a tree."
Quick Answer Template for "A vs B" Questions

1. Identify each model's default flexibility — linear/assumption-heavy = high bias; non-parametric/deep = high variance.

2. Name the knob that controls it — regularization strength, tree depth, k, kernel, number of boosting rounds/epochs.

3. Give the one-liner: "X assumes less structure → more variance; Y assumes more structure → more bias."

All comparisons reflect default/untuned behaviour unless noted; hyperparameters can shift most rows.
