Topic 1

The Core Logic — H₀, H₁, p-value

You have data. You want to make a claim about the world. But data is noisy — how do you know you're seeing a real signal and not random chance? Hypothesis testing gives you a structured framework to answer: "Is this result surprising enough to reject the assumption that nothing is happening?"

H₀ and H₁

H₀ (Null): the default assumption — "nothing interesting is happening." No effect, no difference, no relationship.

H₁ (Alternative): what you're trying to show evidence for.

Example

You change a recommendation algorithm.
H₀: New algo performs the same as old algo (μ_new = μ_old)
H₁: New algo performs differently (μ_new ≠ μ_old)

Key mindset: you never "prove" H₁. You either reject H₀ (data is inconsistent with H₀) or fail to reject H₀ (not enough evidence against it). You never "accept H₀." Absence of evidence ≠ evidence of absence.

The Test Statistic

You reduce your data to a single number measuring how far your observation is from what H₀ predicts.

General Form
test statistic = (observed − expected under H₀) / standard error

Intuitively: how many standard errors away from the null is your observation?

Numerical Example

Sample mean x̄ = 52, H₀ says μ = 50, SE = 2.

z = (52 − 50) / 2 = 1.0 → 1 SE away, not surprising (p ≈ 0.317)
z = (58 − 50) / 2 = 4.0 → 4 SEs away, extremely surprising (p ≈ 0.000063)

The p-value

Definition
p-value = P(data this extreme or more | H₀ is true)

Decision rule: if p < α (typically 0.05), reject H₀.

✗"p-value is the probability that H₀ is true"
✓p-value is the probability of your data (or more extreme), given H₀ is true. P(data | H₀) ≠ P(H₀ | data) — these are completely different things.

The Cookie Jar Intuition

Your mom baked 100 cookies and claims "I didn't eat any" (H₀). You find only 3 left. If her claim were true, this would be extremely hard to explain — that's a low p-value. So you reject her claim. You're not 100% certain she's guilty (maybe a sibling ate them) — you just have strong evidence against the claim.

Cookie StoryStatistics
Mom's claim ("I didn't eat any")H₀ — null hypothesis
Counting missing cookiesCollecting data
"Hard to explain if she's innocent"Low p-value
"I don't believe her"Reject H₀
📌

Key Takeaways: H₀ = default claim, H₁ = what you're trying to show · p-value = P(data this extreme | H₀ true) · reject H₀ when p < α · never "accept H₀," never say p-value = P(H₀ true)

Topic 2

Type I & Type II Errors, Power

When you make a decision (reject or don't reject H₀), you can be wrong in two different ways.

The Courtroom Analogy

Think of H₀ as "defendant is innocent."

You say: Guilty (Reject H₀)You say: Innocent (Fail to reject H₀)
Actually innocent✗ Type I Error✓ Correct
Actually guilty✓ Correct✗ Type II Error
  • Type I Error (α): convicted an innocent person — false alarm / false positive
  • Type II Error (β): let a guilty person go free — missed it / false negative

α is your significance level — setting α = 0.05 means "I'm okay with a 5% chance of a false alarm."

Power = 1 − β

Power = probability of correctly detecting a real effect. If β = 0.20, power = 0.80 → an 80% chance of catching a real signal. Industry standard target: 0.80.

The Tradeoff

Lowering α (stricter threshold) → fewer Type I errors → more Type II errors. They pull against each other. The only way to reduce both at once is more data.

False PositiveFalse Negative
NameType I ErrorType II Error
Symbolαβ
MeaningRejected H₀ wronglyMissed a real effect
Controlled bySignificance levelPower (1−β)

Deep Dive: Deriving the Test Statistic

The formula isn't arbitrary — it comes from standardization (z-scores). Recall z = (x − μ) / σ for a single value. A test statistic does the same for a sample statistic like x̄.

Step 1 — From CLT: x̄ ~ N(μ, σ²/n). Under H₀, μ = μ₀, so x̄ ~ N(μ₀, σ²/n).

Step 2 — Standardize using this distribution's own mean and SD:

z = (x̄ − μ₀) / (σ/√n)

That denominator σ/√n is the standard error — the spread of the sampling distribution, not raw data spread.

Numerical Example

Factory claims μ₀ = 1000 hrs bulb life, σ = 50 (known). Sample n=25, x̄ = 980.

SE = 50/√25 = 10
z = (980 − 1000) / 10 = −2.0
Sample mean is 2 SEs below the claim → p ≈ 0.0455 (two-tailed), borderline significant at α=0.05.

The General Pattern (memorize the skeleton, not just one formula)
test statistic = (estimate − hypothesized value) / SE of the estimate
  • Mean → SE = σ/√n or s/√n
  • Proportion → SE = √(p(1−p)/n)
  • Difference of means (A/B test) → SE = √(σ₁²/n₁ + σ₂²/n₂)

Deep Dive: How α and Power Are Actually Decided

α is not computed — it's a business/risk decision made before the test, based on the cost of a false positive.

ContextTypical αWhy
Drug approval (FDA)0.01 – 0.05False positive harms patients
A/B test, button color0.05 – 0.10Low stakes, fast iteration valued
Particle physics~0.0000003 (5σ)Extraordinary claims need extraordinary evidence
Spam filterTunable, asymmetricBlocking real email is worse than missing spam

Power isn't chosen directly — you set a target (usually 0.80) and solve for required sample size.

Power = f(α, effect size, σ, n)
Increase thisEffect on Power
α (looser threshold)Power ↑
Effect size (bigger true difference)Power ↑
σ (more noise in data)Power ↓
n (more data)Power ↑
Interview-Level Insight

"α is a policy choice about false positive tolerance. Power is what you solve for via sample size, given your α and the smallest effect size you care about detecting."

📌

Key Takeaways: Type I (α) = false positive, Type II (β) = false negative · Power = 1−β, target is usually 0.80 · α↓ causes Power↓ (tradeoff) · only more data improves both · test statistic skeleton: (estimate − hypothesized)/SE

Topic 3

One-Tailed vs Two-Tailed Tests

When you reject H₀, you need to define what counts as "extreme" — too high, too low, or either direction. This is what tails decide.

Two-Tailed Test

H₀: μ = μ₀    H₁: μ ≠ μ₀ — you don't care about direction, just "different." At α=0.05, split 2.5% into each tail. Critical values: ±1.96.

One-Tailed Test

Right-tailed example — H₀: μ ≤ μ₀    H₁: μ > μ₀ — all 5% sits in one tail. Critical value: 1.645.

Numerical Example — Same Data, Different Conclusion

Drug test: μ₀ = 100, z = 1.75.

Two-tailed (H₁: μ≠100): critical = ±1.96 → 1.75 < 1.96 → fail to reject
One-tailed (H₁: μ>100): critical = 1.645 → 1.75 > 1.645 → reject H₀

Same data, same z — opposite conclusions. Direction must be decided before seeing data.

✗"I'll look at the data first, see which direction it went, then pick a one-tailed test in that direction to make my result significant."
✓This is p-hacking. Picking direction after seeing results means your real false-positive rate is no longer the α you claim.
Use Two-Tailed WhenUse One-Tailed When
You care about any difference (most A/B tests)Strong pre-registered directional hypothesis
Default/safe choice in industryRare, viewed skeptically by reviewers
Conservative — needs more evidenceEasier to reach significance

Deep Dive: How Critical Values Are Calculated

The critical value is the z (or t) score that cuts off exactly α of probability in the tail(s). You work backward from a probability to a score using the inverse CDF.

Test TypeTail AreaCritical Value (α=0.05)
Two-tailed0.025 each side±1.96
Right-tailed0.05 on right+1.645
Left-tailed0.05 on left−1.645
from scipy.stats import norm # Two-tailed, alpha = 0.05 z_two_tailed = norm.ppf(1 - 0.05/2) # → 1.96 # One-tailed, alpha = 0.05 z_one_tailed = norm.ppf(1 - 0.05) # → 1.645 # t-test version (needs degrees of freedom) from scipy.stats import t t_critical = t.ppf(1 - 0.05/2, df=24) # for n=25 sample

t-critical values are always slightly larger than z-critical for the same α (fatter tails) — the gap shrinks as n grows, converging to z as df → ∞.

Intuition for Stricter α

α=0.01 means accepting only a 1% false-positive chance. To carve out a smaller sliver of tail area, you must go further out — the curve is thin out there, so more distance is needed to capture less area. Stricter α → larger critical value in magnitude.

📌

Key Takeaways: Two-tailed splits α across both tails; one-tailed concentrates it in one → smaller critical value, easier to reject · direction must be pre-registered or it's p-hacking · critical values come from inverse CDF (norm.ppf / t.ppf) · industry default is two-tailed

Topic 4

Common Tests — z, t, χ², F

Every test is the same skeleton — (estimate − hypothesized)/SE — applied to a different situation. Choice depends on data type, what's being compared, and whether σ is known.

1. Z-Test

When: testing a mean/proportion, population σ known (or n≥30 so s≈σ).

z = (x̄ − μ₀) / (σ/√n)
Numerical Example

σ=40 (known), n=64, x̄=990, μ₀=1000.
SE = 40/√64 = 5 → z = (990−1000)/5 = −2.0 → p≈0.0455 → reject H₀.

True z-tests are rare in practice — you almost never know population σ. Mostly used for proportions.

2. T-Test

When: testing a mean, σ unknown (the normal real-world case) — estimate with sample s.

t = (x̄ − μ₀) / (s/√n),   df = n − 1

The t-distribution has fatter tails than normal to account for the extra uncertainty of estimating σ. As n grows, t → normal.

TypeUse CaseExample
One-sampleCompare sample mean to fixed value"Is avg delivery time ≠ 30 mins?"
Independent two-sampleCompare means of two separate groupsA/B test mean comparison
PairedSame subjects, before/afterWeight before vs after diet
Independent Two-Sample (A/B test core formula)
t = (x̄₁ − x̄₂) / √(s₁²/n₁ + s₂²/n₂)
Numerical Example — Paired Test

5 people's weight diff before/after diet: [−2,−1,−3,0,−2] kg

mean diff (d̄) = −1.6, s_d ≈ 1.14, SE ≈ 0.51
t = −1.6/0.51 ≈ −3.14, df=4 → critical t ≈ ±2.78 → |−3.14|>2.78 → reject H₀, diet had a real effect.

Paired beats independent here because differencing per-person cancels person-to-person variation — same logic as variance reduction.

3. Chi-Square Test (χ²)

When: data is categorical. Two cases: (a) Goodness of Fit — does observed match expected? (b) Independence — are two categorical variables related?

χ² = Σ (Observed − Expected)² / Expected
Numerical Example — Goodness of Fit

Roll a die 60 times, expect 10/face.

FaceObsExp(O−E)²/E

χ² = 4.4, df=5 → critical χ²≈11.07 → 4.4<11.07 → fail to reject, die looks fair.

4. F-Test

When: comparing variances. Basis of ANOVA and the equal-variance check before a t-test.

F = s₁² / s₂²   (larger variance on top)
Numerical Example

Group A var=25, Group B var=10 → F = 25/10 = 2.5. Compare against critical F (depends on both df's).

ANOVA's F-statistic asks: "is between-group variance large relative to within-group variance?" — same signal/noise logic throughout this entire subtopic.

✗"Conversion rate" sounds continuous because it's a percentage, so I'll use a t-test.
✓Look at the raw underlying data: each user converted (1) or not (0) — that's categorical. Use chi-square (or an equivalent two-proportion z-test), not a t-test.

Degrees of Freedom (df)

df = the number of values free to vary, once constraints are fixed. Example: 5 numbers with a fixed mean of 10 (sum=50) — you can freely pick 4, the 5th is forced. 5 numbers, 4 degrees of freedom.

This is why sample variance uses (n−1): s² = Σ(xᵢ−x̄)²/(n−1) — computing x̄ already used up 1 degree of freedom.

Why it matters: lower df → fatter tails → need a more extreme statistic to reach significance.

from scipy.stats import t t.ppf(0.975, df=4) # ≈ 2.776 (small sample, fat tails) t.ppf(0.975, df=100) # ≈ 1.984 (large sample, ≈ normal)
📌

Key Takeaways: z-test (σ known, rare) vs t-test (σ unknown, common) · 3 t-test flavors: one-sample, independent, paired · chi-square for categorical data · F-test compares variances, basis of ANOVA · df = "values free to vary," costs 1 per estimated parameter

Topic 5

Assumptions

Every test has fine print — assumptions that must hold for p-values and critical values to be valid. Violate them, and "p = 0.03" can be meaningless.

1. Normality

✗"T-tests require my raw data to be normally distributed."
✓T-tests require the sampling distribution of the mean to be approximately normal — CLT makes this hold even with non-normal raw data, given reasonably large n.

Breaks when: small n (<30ish) with heavily skewed/heavy-tailed raw data — CLT hasn't kicked in yet.

Check: Q-Q plot, Shapiro-Wilk test, histogram shape. Fix: larger n, log-transform skewed data, or use a non-parametric test (e.g. Mann-Whitney U).

2. Independence

Each observation should be unrelated to others. Violation example: the same user appearing multiple times in an A/B test, counted as separate "independent" rows.

Why it matters: violating independence artificially shrinks SE, inflating your test statistic and your true false-positive rate beyond stated α.

Common violations: repeated measurements per user, time-series autocorrelation, clustered data (students in a classroom).

Fix: aggregate to one row per independent unit, use mixed-effects models / clustered SEs, or autocorrelation-aware models for time series.

3. Homoscedasticity (Equal Variance)

Groups being compared should have similar spread, not just similar means.

Numerical Example

Group A (control) var=10, Group B (new feature) var=200. Even with similar means, B's huge variance distorts the pooled SE estimate → unreliable test statistic.

Check: Levene's test, the F-test (F=s₁²/s₂²), side-by-side boxplots. Fix: Welch's t-test — doesn't assume equal variance, adjusts df automatically. Many statisticians default to Welch's always.

AssumptionWhat BreaksDetectionFix
NormalitySkewed/small-n dataQ-Q plot, Shapiro-WilkMore data, transform, non-parametric test
IndependenceRepeated/clustered/time-dependent dataDomain logic, Durbin-WatsonAggregate per unit, mixed models
HomoscedasticityUnequal variance between groupsLevene's test, F-testWelch's t-test
Interview-Level Insight

Independence is the assumption that's hardest to fix after the fact and most likely to silently inflate your false positive rate. Normality is usually the least dangerous violation because CLT bails you out with reasonable sample sizes. Danger ranking: independence > homoscedasticity > normality — many candidates get this backward.

📌

Key Takeaways: Normality: CLT often saves you, least dangerous · Independence: inflates false positives by shrinking SE — most dangerous, hardest to fix · Homoscedasticity: unequal variance distorts pooled SE; Welch's t-test is the safe default

Topic 6

Statistical vs Practical Significance

You can get p = 0.001 for an effect that's completely useless in the real world. p-values alone are dangerously incomplete.

Statistical significance answers: "Is this effect likely real, or just noise?" (the p-value question)
Practical significance answers: "Does this effect actually matter?" (a business/domain question)

The Trap: Large n Makes Everything "Significant"

Since z = (x̄−μ₀)/(σ/√n), as n grows, SE shrinks toward 0 — even a meaningless difference eventually produces a tiny p-value.

Numerical Example

Website redesign A/B test: control 10.00% conversion, treatment 10.01%.

n=100/group → p=0.85 (not significant, as expected)
n=10,000,000/group → p=0.001 (statistically significant!)

The 0.01 percentage point difference never changed — only n changed.

Effect Size — Cohen's d

d = (x̄₁ − x̄₂) / s_pooled
dInterpretation
0.2Small effect
0.5Medium effect
0.8Large effect
Numerical Example

Mean difference = 5, pooled std = 10 → d = 5/10 = 0.5 — medium effect, even if p<0.001.

Effect Size for Proportions

Absolute lift = p_treatment − p_control
Relative lift = (p_treatment − p_control) / p_control

10.00%→10.01% is a 0.01pp absolute lift, or 0.1% relative lift. No reasonable business ships for that, even with p<0.001 — implementation cost likely exceeds the value.

Interview-Level Insight

p-value tells you the result is unlikely to be noise. Effect size tells you whether it's big enough to act on. Always report both — at internet scale, n is huge and everything becomes "significant."

Confidence Intervals Bridge Both

Instead of just a p-value, report: "Conversion increased by 0.01% [CI: 0.005% to 0.015%]" — communicates significance (excludes 0?) and magnitude (how big, with uncertainty) in one number.

ConceptAnswersFormula/Metric
Statistical significanceIs it real?p-value
Practical significanceDoes it matter?Effect size, lift %
Both at onceReal AND how big, with uncertaintyConfidence interval
📌

Key Takeaways: Large n → tiny effects become "significant" (expected, not a flaw) · effect size tells magnitude, p-value tells reality · always report both · decision to ship should weigh effect size against cost, not just p<0.05

Topic 7

Multiple Testing Problem

Every test has some chance of a false positive (α, typically 5%). Fine for one test. But run many tests, and by pure chance you'll likely see false "wins."

The Math

P(at least one false positive) = 1 − (1 − α)^m
Numerical Example

m=20 tests, α=0.05 → 1 − (0.95)^20 = 1 − 0.358 = 0.642

64% chance of at least one false "significant" result — even if nothing real exists anywhere. This is the Family-Wise Error Rate (FWER).

This is exactly how p-hacking disasters happen: testing 20 metrics, reporting only the one significant result.

Fix 1: Bonferroni Correction

α_corrected = α / m
Numerical Example

m=20, want overall α=0.05 → α_corrected = 0.05/20 = 0.0025. Each test now needs p<0.0025.

Trade-off: very conservative — dramatically reduces power (more Type II errors). At m=100, bar becomes p<0.0005.

Fix 2: False Discovery Rate (Benjamini-Hochberg)

Instead of "probability of even one false positive" (strict), control the expected proportion of false positives among declared discoveries.

FDR = E[false positives / total positives declared]

BH procedure:

  1. Rank all m p-values smallest to largest
  2. Find largest k such that p₍ₖ₎ ≤ (k/m) × α
  3. Reject H₀ for all tests with rank ≤ k
Numerical Example

m=5, α=0.05, sorted p-values: 0.001, 0.008, 0.02, 0.04, 0.12

Rank (k)p-valueThreshold (k/m × α)Pass?
10.0010.01✓
20.0080.02✓
30.020.03✓
40.040.04✓ (equal, passes)
50.120.05✗

Largest k where it passes = 4 → reject H₀ for ranks 1–4.

Bonferroni (FWER)Benjamini-Hochberg (FDR)
ControlsP(any false positive)Expected % false positives among discoveries
StrictnessVery conservativeLess conservative, more power

Interview-Level Insight

If you're running a single confirmatory test, you don't need correction. The moment you run multiple comparisons and report "the significant one," you must correct — otherwise your stated α is a lie. Use Bonferroni when false positives are very costly (few tests); use FDR/BH when scanning many metrics and wanting power.

📌

Key Takeaways: Testing many things inflates false-positive probability (FWER) · Bonferroni: α/m, conservative, low power · BH/FDR: controls proportion of false discoveries, more power · always correct when reporting "the significant one" from many tests

Topic 8

Interview Traps & Common Misconceptions

Rapid-fire consolidation of every trap embedded across Subtopics 1–7 — exactly how interviewers test if you actually understand this or just memorized formulas.

Trap 1 — p-value misinterpretation

✗p = 0.03 means 3% chance H₀ is true
✓p = P(data this extreme | H₀ true)

Trap 2 — "Accepting" the null

You can only "fail to reject" — absence of evidence ≠ evidence of absence. Maybe your test was just underpowered.

Trap 3 — Confusing α with p-value

α is the threshold set beforehand (a policy decision). p-value is computed from data. They're compared, not the same thing.

Trap 4 — One-tailed test chosen after seeing data

Direction must be pre-registered, or it's p-hacking and your real α is no longer what you claim.

Trap 5 — "n=30 always makes things normal"

False — CLT convergence speed depends on how skewed/heavy-tailed the original distribution is. Highly skewed data may need much larger n.

Trap 6 — Statistical significance = practical significance

Large n makes trivial effects "significant." Always report effect size alongside p-value.

Trap 7 — Running multiple tests without correction

Testing 20 metrics and reporting the 1 significant one without correction is implicit p-hacking.

Trap 8 — Peeking at A/B results repeatedly and stopping early

Checking p-value every day and stopping as soon as it dips below 0.05 inflates false positive rate dramatically — "optional stopping," a special case of the multiple testing problem (each peek is another test).

Trap 9 — Confusing correlation in test statistics with causation

A significant t-test on observational data tells you means differ — it says nothing about why.

Trap 10 — Assuming higher n always fixes a bad test design

More n fixes power problems (Type II error) but does nothing for violated independence or biased sampling. Garbage data at scale is still garbage.

Final

30 Hypothesis Testing Interview Questions

Q1Easy
What is a null hypothesis?
+

The default assumption that nothing interesting is happening — no effect, no difference, no relationship. It's the claim you're trying to find evidence against.

Q2Easy
What does a p-value actually represent?
+

The probability of observing data this extreme (or more extreme), assuming H₀ is true. It is NOT the probability that H₀ is true.

Q3Easy
What's the difference between H₀ and H₁?
+

H₀ is the default/no-effect claim. H₁ is what you're trying to find support for — typically the presence of an effect or difference.

Q4Easy
Why do we say "fail to reject H₀" instead of "accept H₀"?
+

Failing to find evidence against H₀ doesn't prove H₀ is true — your test might just be underpowered. Absence of evidence is not evidence of absence.

Q5Easy
What is a Type I error? What is a Type II error?
+

Type I (α): rejecting H₀ when it's actually true — a false positive. Type II (β): failing to reject H₀ when it's actually false — a false negative.

Q6Easy
What is statistical power?
+

Power = 1 − β = the probability of correctly detecting a real effect when one exists. Industry standard target is 0.80.

Q7Easy
What's the difference between a one-tailed and two-tailed test?
+

Two-tailed tests for a difference in either direction (H₁: μ≠μ₀), splitting α across both tails. One-tailed tests for a difference in one specific direction (H₁: μ>μ₀ or μ<μ₀), putting all of α in one tail — making it easier to reject H₀.

Q8Easy
When would you use a t-test instead of a z-test?
+

When the population standard deviation σ is unknown and must be estimated from the sample (s). This is the common real-world case. Z-tests require known σ.

Q9Easy
What does "degrees of freedom" mean intuitively?
+

The number of values free to vary once certain constraints (like a fixed mean) are set. E.g., 5 numbers with a fixed mean — 4 are free to vary, the 5th is forced. Estimating a parameter from data costs 1 degree of freedom.

Q10Easy
What significance level (α) is most commonly used, and what does it mean?
+

α = 0.05 is most common — it means you're willing to accept a 5% chance of a false positive (rejecting H₀ when it's actually true).

Q11Medium
Derive the general form of a test statistic from first principles.
+

From CLT, x̄ ~ N(μ, σ²/n). Under H₀, μ=μ₀, so x̄ ~ N(μ₀, σ²/n). Standardizing this (just like z=(x−μ)/σ for a single value) gives z=(x̄−μ₀)/(σ/√n). The denominator is the standard error — the spread of the sampling distribution, not raw data.

Q12Medium
Why does lowering α increase the risk of Type II errors?
+

Lowering α makes the rejection threshold stricter — you need stronger evidence to reject H₀. This means real effects are more likely to be missed (failed to reject when H₀ is actually false), increasing β and decreasing power.

Q13Medium
Explain the three types of t-tests and when each is used.
+

One-sample: compare a sample mean to a fixed/known value. Independent two-sample: compare means of two separate, unrelated groups (e.g., A/B test). Paired: compare two measurements on the same subjects (e.g., before/after), which cancels out subject-level variation and increases power.

Q14Medium
A categorical variable like "converted or not" is being analyzed for an A/B test. Why is a t-test the wrong choice?
+

Because the raw underlying data is binary/categorical (1=converted, 0=not), not continuous — even though the summary statistic (conversion rate) looks like a percentage. The correct test is chi-square (or an equivalent two-proportion z-test).

Q15Medium
What's the practical consequence of violating the independence assumption?
+

It artificially shrinks the standard error, inflating the test statistic and making results look more significant than they really are — your true false-positive rate ends up higher than your stated α.

Q16Medium
Why is the t-distribution's tail fatter than the normal distribution's?
+

Because we're estimating σ with the sample statistic s, which adds extra uncertainty. The fatter tails account for that — as n grows, s becomes a better estimate of σ, and the t-distribution converges to normal.

Q17Medium
What's the formula for an independent two-sample t-test, and how does it connect to A/B testing?
+

t = (x̄₁−x̄₂) / √(s₁²/n₁ + s₂²/n₂). This is literally the formula behind comparing two group means in an A/B test — same "(difference)/SE" skeleton, with SE combining variance from both groups.

Q18Medium
An A/B test shows p=0.001 with a 0.05% relative lift on 50 million users. Should you ship it?
+

Not based on p-value alone. With huge n, even trivial effects become "statistically significant." You need effect size / absolute business impact, confidence interval width, cost of shipping, and any secondary metric impact before deciding — statistical significance ≠ practical significance.

Q19Medium
What is the Bonferroni correction and what's its main drawback?
+

α_corrected = α/m, where m is the number of tests. It controls the family-wise error rate (probability of even one false positive). Drawback: very conservative, dramatically reduces power as m grows — you'll miss real effects.

Q20Medium
What's the difference between FWER and FDR?
+

FWER (Bonferroni) controls the probability of even one false positive across all tests — very strict. FDR (Benjamini-Hochberg) controls the expected proportion of false positives among declared "discoveries" — less strict, more statistical power, better suited to many simultaneous tests (e.g., genomics, dozens of A/B metrics).

Q21Hard
Why is "peeking" at A/B test results daily and stopping as soon as p<0.05 a statistical error?
+

It's a form of the multiple testing problem ("optional stopping") — each peek is effectively another test, so your real cumulative false-positive rate is much higher than the stated α. Over many days of peeking, you're almost guaranteed to hit a spurious p<0.05 at some point even with no real effect. Fix: pre-register a fixed sample size/duration, or use sequential testing methods designed for repeated looks (e.g., alpha-spending functions).

Q22Hard
You're comparing variances between two groups before deciding whether to use Student's or Welch's t-test. Walk through how you'd test this and what you'd conclude from F=2.5.
+

Use an F-test: F=s₁²/s₂² (larger variance on top). Compare F against a critical F-value from the F-distribution with both groups' df. If F exceeds the critical value, variances are significantly different → use Welch's t-test (doesn't assume equal variance, adjusts df). If not, Student's t-test is fine. In practice, many statisticians default to Welch's regardless, since it costs little when variances are actually equal.

Q23Hard
Why does CLT "rescue" the normality assumption for t-tests but not fix violated independence?
+

CLT guarantees the sampling distribution of the mean approaches normal as n grows, regardless of the raw data's shape — this directly satisfies the normality assumption. But CLT says nothing about whether observations are independent; it assumes independence as a precondition. No amount of additional (correlated) data fixes a violated independence assumption — you need to restructure the analysis (e.g., aggregate per independent unit).

Q24Hard
Derive why df = n−1 for a one-sample t-test using the constraint argument.
+

Computing s² requires first computing x̄ from the same data. Once x̄ is fixed, only n−1 of the n deviations (xᵢ−x̄) are free to vary — the last one is mathematically forced so that the deviations sum to zero (a property of the mean). This "using up" one piece of information is why we divide by (n−1), not n, in the sample variance formula.

Q25Hard
A team runs 20 independent metric tests in an A/B experiment, all at α=0.05, with no real effects anywhere. What's the probability they see at least one false positive, and what should they do about it?
+

P(at least one FP) = 1−(1−0.05)^20 ≈ 0.64 — a 64% chance of a spurious "win" purely by chance. They should apply a multiple-testing correction: Bonferroni (α/20=0.0025 per test) if false positives are very costly, or Benjamini-Hochberg if they want more power and can tolerate a controlled fraction of false discoveries among reported wins.

Q26Hard
Explain why choosing a one-tailed test after looking at your data inflates your real false-positive rate, even if you still report α=0.05.
+

A one-tailed test concentrates the entire rejection region in one direction, with a smaller critical value (e.g., 1.645 vs 1.96) than a two-tailed test. If you choose the direction post-hoc based on which way the data happened to lean, you're implicitly able to "reject" in either direction depending on outcome — giving yourself two one-tailed shots instead of committing to one. Your effective false-positive rate becomes closer to 2×α, even though you report α=0.05.

Q27Hard
Your A/B test shows a statistically significant negative result on a secondary metric you didn't pre-register, found while exploring 15 different cuts of the data. How should you treat this finding?
+

Treat it as a hypothesis-generating signal, not a confirmed result — this is exactly the multiple testing problem (15 implicit comparisons inflate false-positive risk to roughly 1−0.95^15 ≈ 54%). The correct move is to apply a multiple testing correction retroactively, or better, run a dedicated, pre-registered follow-up test specifically on this metric before drawing conclusions or taking action.

Q28Hard
Why might increasing sample size fail to "fix" a non-significant A/B test result, and how would you distinguish a true null effect from an underpowered test?
+

If the true effect size is genuinely zero or negligible, no amount of additional data will make it significant in a meaningful way (or it will, but the effect will be too small to matter practically). To distinguish: run a power analysis using the smallest effect size you'd care about (MDE) — if your current n already has 80%+ power to detect that MDE and you're still not significant, that's evidence for a true null. If power is low, the test result is inconclusive, not evidence of "no effect."

Q29Hard
Why does a 2×2 contingency table for a conversion A/B test have df=1, and what does that practically mean?
+

df = (rows−1)(columns−1) = (2−1)(2−1) = 1. Practically: once you know the row totals, column totals, and any one cell's value, all other 3 cells are mathematically determined. This is also why a 2×2 chi-square test is mathematically equivalent to a two-proportion z-test — both reduce to comparing one piece of independent information.

Q30Hard
You're asked: "Our test is statistically significant with p=0.04, should we trust it?" What's your full checklist before answering?
+

(1) Was this the only test run, or one of many — does it need multiple-testing correction? (2) Was the test direction (one/two-tailed) pre-registered? (3) Are the assumptions (independence, especially) actually satisfied — any repeated units or autocorrelation? (4) What's the effect size — is it practically meaningful, not just statistically detectable? (5) Was the sample size/stopping point pre-determined, or was this found via repeated peeking? (6) Does the confidence interval support a consistent, actionable conclusion? A single p-value in isolation answers none of these — full context matters more than the number itself.
